# 作用
可以实时监控或者后台记录运行时的性能

可以与事件管理器结合，实现超过某一个阈值后输出日志

可以与计划管理器结合，超过某一个阈值后执行某一个脚本

# 1 cpu
## 1.1  processor
可选择指定cpu核心，

<font style="color:rgb(51, 51, 51);">CPU利用率（% Processor Time）：</font>

<font style="color:rgb(51, 51, 51);">指处理器执行非闲置线程时间的百分比</font>

<font style="color:rgb(51, 51, 51);">中断率（Interrupts/sec.）：</font>

<font style="color:rgb(51, 51, 51);">每秒钟设备中断处理器的次数。取决于处理器，越低越好；不宜超过1,000；</font>

<font style="color:rgb(34, 34, 34) !important;">Processor：%Processor Time （常用）  
</font><font style="color:rgb(34, 34, 34) !important;"> 如果CPU使用率的值持续超过95%，则表示CPU是瓶颈。  
</font><font style="color:rgb(34, 34, 34) !important;"> 计算方式：%Processor Time值 = 100% - Idle process时间比例(即 空闲线程Idle Thread)，</font>

<font style="color:rgb(34, 34, 34) !important;"> Processor：% User Time  
</font><font style="color:rgb(34, 34, 34) !important;"> 是系统非核心操作消耗的CPU时间；  
</font><font style="color:rgb(34, 34, 34) !important;"> 如果表示数据库，则% User Time值大的原因可能是数据库的排序或者函数操作导致消耗过多的CPU时间；</font>

<font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">系统中的处理器队列长度（Processor Queue Length）：表示在某个时间点上正在等待执行的线程数量。较高的队列长度表明有较多的线程正在等待CPU资源，这可能是CPU繁忙或过载的指</font>
