结合 CodeGraph + Claude Code 的协作逻辑，分通用话术、查代码关系、改代码、排查问题、大型重构几类，直接复制就能用，同时区分简单场景和复杂场景。
 
前提：你已经执行  codegraph init -i && codegraph index . --watch ，索引和监听正常运行。
 
 
 
一、通用开场（每次会话首选，让 Claude 主动用 CodeGraph）
 
短句版（日常快速用）
 
1. 本项目已配置 CodeGraph 代码图谱，分析代码、梳理调用关系、查看依赖请优先调用工具查询，不要逐文件翻找。
2. 借助 CodeGraph 分析项目代码结构与符号关系，基于图谱信息回答和执行任务。
 
完整版（新项目/复杂项目，一次性说明）
 
项目已完成 CodeGraph 索引并开启实时监听，所有代码调用链、类继承、模块导入、函数依赖都已生成图谱。请优先使用 CodeGraph 查询代码关系、定位上下游逻辑、梳理架构，减少手动遍历文件。
 
 
 
二、按场景细分话术（直接复制使用）
 
场景1：梳理整体架构/目录/模块依赖
 
1. 用 CodeGraph 梳理整个项目的模块依赖和核心调用链路，简述整体架构。
2. 借助代码图谱，列出核心类、入口函数以及各模块之间的引用关系。
 
场景2：查询单个函数/类（高频）
 
1. 用 CodeGraph 查找  xxxFunction  函数的所有调用方、被调用方，梳理完整调用链。
2. 通过代码图谱分析  XXXClass  的继承关系、实现接口以及外部引用位置。
3. 帮我定位  xxx()  方法在项目里所有出现的位置和调用场景。
 
场景3：阅读/理解某段逻辑
 
1. 先通过 CodeGraph 理清当前文件上下游依赖，再逐行解读这段代码逻辑。
2. 结合代码图谱，分析该功能的执行链路、数据流向。
 
场景4：修改代码/改函数签名/增删方法（重点，防漏改）
 
1. 修改这个函数，先用 CodeGraph 查出所有调用处，逐一评估影响范围，再完成修改。
2. 我要调整  xxx  方法入参，先通过代码图谱梳理全部调用方，统一适配修改。
3. 删除这个工具方法前，用 CodeGraph 检查项目内是否还有未清理的引用。
 
场景5：排查 Bug/异常链路
 
1. 接口出现报错，先用 CodeGraph 梳理从入口到报错位置的完整调用链，定位问题节点。
2. 分析该异常的传播路径，借助代码图谱找到所有关联逻辑。
 
场景6：功能重构/模块拆分/代码优化
 
1. 基于 CodeGraph 的依赖图谱，评估当前模块的耦合度，给出重构方案。
2. 梳理该模块对外暴露的接口与外部依赖，设计拆分方案。
 
场景7：跨文件/跨模块溯源
 
1. 追踪这个变量/常量的定义位置、所有引用场景，用代码图谱完整溯源。
 
 
 
三、进阶约束话术（规范 Claude 行为，避免低效操作）
 
适合大型项目、代码量多的时候加在指令里：
 
1. 优先使用 CodeGraph 代码图谱查询关系，不要全盘遍历项目文件，提升效率。
2. 分析代码影响范围时，必须基于图谱结果，不要主观猜测调用关系。
3. 代码有变更后，依赖 CodeGraph 自动更新的图谱，同步校验引用是否正常。
