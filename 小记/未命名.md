Windows 系统 OpenClaw 安装手册（2026 最新）
 
一、系统与环境要求
 
- 操作系统：Windows 10 64位 / Windows 11 64位
- 核心依赖：Node.js ≥ v22.0.0（LTS 推荐）
- 内存/磁盘：≥2GB 内存、≥500MB 可用磁盘
- 可选：Git（源码安装）、Docker（容器部署）、Python 3.10+（部分技能） 
 
二、安装方案总览（4种）
 
方案 优点 缺点 适用人群 
方案1：一键脚本（PowerShell） 极简、自动配环境、零手动依赖 需管理员权限、网络敏感 新手/快速部署 
方案2：npm 全局安装（原生） 轻量、无虚拟化、上手快 稳定性一般、服务配置复杂 临时测试/轻量使用 
方案3：WSL2 安装（官方推荐） 兼容性最强、支持完整服务、稳定 需开启虚拟化、懂基础 Linux 长期使用/生产环境 
方案4：Docker 容器安装 环境隔离、一键卸载、无依赖冲突 资源占用高、需 Docker 基础 多环境隔离/开发测试 
 
三、方案1：一键脚本安装（PowerShell，新手首选）
 
步骤1：管理员身份运行 PowerShell
 
- 开始菜单搜索 PowerShell → 右键 以管理员身份运行
 
步骤2：执行一键安装命令（自动装 Node.js+OpenClaw）
 
powershell
  

curl -fsSL https://openclaw.ai/install.cmd -o install.cmd && install.cmd --tag latest && del install.cmd
 
 
- 说明： --tag latest  装稳定版；换  --tag beta  装预览版 
- 等待脚本执行（约3–5分钟，自动配置环境）
 
步骤3：启动与验证
 
- 安装完成后自动弹出 Web UI（默认 http://localhost:3000）
- 终端输入  openclaw --version ，输出版本号即成功 
 
四、方案2：npm 全局安装（原生 Windows）
 
步骤1：安装 Node.js 22.x
 
1. 官网下载：https://nodejs.org/zh-cn/ → Windows 64位  .msi （LTS） 
2. 双击安装 → 全程默认路径 → 完成
 
步骤2：验证 Node.js
 
powershell
  

node --version   # 需 v22.x.x
npm --version    # 需 10.x.x+
 
 
步骤3：配置 npm 镜像（国内加速，可选但推荐）
 
powershell
  

npm config set registry https://registry.npmmirror.com/
 
 
步骤4：全局安装 OpenClaw
 
powershell
  

# 稳定版
npm install -g openclaw@latest --verbose
# 或预览版
npm install -g openclaw@beta --verbose
 
 
-  --verbose  显示详细日志，避免卡死无反馈
 
步骤5：启动与初始化
 
powershell
  

# 启动引导配置
openclaw init
# 启动服务（默认 3000 端口）
openclaw start
 
 
- 浏览器访问 http://localhost:3000 完成初始化配置 
 
五、方案3：WSL2 安装（官方推荐，最稳定）
 
步骤1：安装 WSL2（管理员 PowerShell）
 
powershell
  

# 一键启用 WSL2 + 安装 Ubuntu
wsl --install -d Ubuntu
 
 
- 重启电脑 → 自动弹出 Ubuntu 终端 → 设置用户名/密码（无需管理员） 
 
步骤2：WSL2 启用 systemd（关键，支持后台服务）
 
1. Ubuntu 终端执行：
 
bash
  

sudo tee /etc/wsl.conf > /dev/null << 'EOF'
[boot]
systemd=true
[interop]
enabled=true
appendWindowsPath=true
EOF
 
 
2. PowerShell 重启 WSL：
 
powershell
  

wsl --shutdown
 
 
3. 重新进入 WSL，验证： systemctl is-active --quiet && echo "systemd 已启用"  
 
步骤3：WSL 内安装 Node.js 22.x
 
bash
  

curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash - && sudo apt install -y nodejs
# 验证
node --version
npm --version
 
 
步骤4：WSL 内 npm 安装 OpenClaw
 
bash
  

sudo npm install -g openclaw@latest --verbose
 
 
步骤5：启动服务（WSL 内）
 
bash
  

# 初始化
openclaw init
# 启动（后台守护进程）
sudo systemctl start openclaw
sudo systemctl enable openclaw  # 开机自启
 
 
步骤6：Windows 访问 Web UI
 
- WSL 内查 IP： hostname -I （如 192.168.1.100）
- Windows 浏览器访问：http://192.168.1.100:3000
 
六、方案4：Docker 容器安装（环境隔离）
 
步骤1：安装 Docker Desktop
 
1. 下载：https://www.docker.com/products/docker-desktop/
2. 安装时勾选 WSL 2 集成 → 完成后启动 Docker
 
步骤2：拉取 OpenClaw 镜像
 
powershell
  

# 拉取最新镜像
docker pull openclaw/openclaw:latest
 
 
步骤3：创建并启动容器
 
powershell
  

docker run -d \
  --name openclaw \
  -p 3000:3000 \
  -v /var/lib/openclaw:/app/data \
  --restart always \
  openclaw/openclaw:latest
 
 
- 端口映射：3000（主机）→3000（容器）
- 数据持久化： /var/lib/openclaw  挂载容器数据目录
 
步骤4：访问 Web UI
 
- 浏览器打开 http://localhost:3000 → 完成初始化配置
 
七、常见问题与解决
 
1. Node.js 版本过低
 
- 错误： requires Node.js >=22.0.0 
- 解决：卸载旧版 → 安装 Node.js 22.x LTS → 重启终端
 
2. 端口 3000 被占用
 
- 错误： EADDRINUSE: address already in use :::3000 
- 解决：- 查占用进程： netstat -ano | findstr :3000  → 结束进程
- 或自定义端口： openclaw start --port 3001  → 访问 http://localhost:3001 
 
3. Powershell 执行策略限制
 
- 错误：禁止运行脚本
- 解决（管理员）：
 
powershell
  

Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
 
 
4. 安装时依赖下载慢/失败
 
- 解决：切换 npm 镜像为国内源
 
powershell
  

npm config set registry https://registry.npmmirror.com/
 
 
5. WSL2 网络不通
 
- 解决：重启 WSL
 
powershell
  

wsl --shutdown
 
 
八、卸载方法
 
方案1/2（npm 安装）
 
powershell
  

npm uninstall -g openclaw
 
 
方案3（WSL2 安装）
 
bash
  

sudo npm uninstall -g openclaw
 
 
方案4（Docker 安装）
 
powershell
  

# 停止并删除容器
docker stop openclaw
docker rm openclaw
# 删除镜像（可选）
docker rmi openclaw/openclaw:latest
