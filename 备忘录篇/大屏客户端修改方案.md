## 1. 新增配置文件
在软件根目录的 `config` 文件夹下新增配置文件 `seres_custom_config.json`，内容格式如下：

```json
{
    "is_enable": false,
    "station": [
        {
            "product_prefix": "",
            "ip": "127.0.0.1"
        },
        {
            "product_prefix": "",
            "ip": "127.0.0.1"
        }
    ]
}
```

## 2. 接口改造
针对大屏向检测站获取记录的接口 `ScreenDefectHandle::getInspectRecordsAndDefectRecords`，使用**策略模式**进行封装。

策略模式相关代码文件应生成在以下路径：  
`worker_screen_client\src\handles\inspect_record_access\`

### 2.1 旧逻辑（默认行为）
满足以下任一条件时，沿用原有逻辑：

+ 配置文件不存在
+ 配置文件中 `is_enable` 为 `false`

### 2.2 定制功能（`is_enable` 为 `true` 时）
#### 2.2.1 生成两个查询用的 VIN
1. 计算配置文件中 `product_prefix` 字符串长度 `n`
2. 将 `VehicleOnPosParam::vin` 的前 `n` 个字符删除，生成新的 VIN（称为 **基础VIN**）
3. 使用配置文件中的多个 `product_prefix` 分别与基础VIN拼接，生成两个 VIN：
    - `vin1` = `product_prefix1` + 基础VIN
    - `vin2` = `product_prefix2` + 基础VIN

#### 2.2.2 获取检测记录列表
通过以下方式分别向两个检测站获取检测记录：

```cpp
ProjectDataMgr::instance()
    .getSpecularVisionGenMgr()
    ->getSpecularVisionGenByIP(配置中的ip)
```

获取 RPC 对象后，调用接口：

```cpp
QueryOperatorImpl::getInspectedVehicleList(VIN)
```

每个检测站分别使用对应的 `ip` 和生成的 VIN（vin1 / vin2）进行调用。

最终可获取：

+ 检测站1的所有指定产品ID的检测记录列表
+ 检测站2的所有指定产品ID的检测记录列表

#### 2.2.3 过滤检测记录
编写一个过滤函数，输入为上述两个检测记录列表，过滤逻辑如下：

+ 对于相同 `InspectedVehicleRecord::vehicleModelName` 名称的记录，**只保留InspectedVehicleRecord::time时间最新的记录**
+ 理论上，过滤后最多输出 **2 条**检测记录

#### 2.2.4 获取缺陷记录
根据上一步过滤后返回的检测记录，分别向对应的 RPC 对象中获取缺陷记录。
