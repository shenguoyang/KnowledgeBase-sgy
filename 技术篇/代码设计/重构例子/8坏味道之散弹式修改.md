### <font style="color:rgb(64, 64, 64);">代码坏味道：</font>**<font style="color:rgb(64, 64, 64);">散弹式修改（Shotgun Surgery）</font>**
#### <font style="color:rgb(64, 64, 64);">定义</font>
**<font style="color:rgb(64, 64, 64);">散弹式修改</font>**<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">是指当需要对某个功能进行修改时，必须在多个类或文件中进行大量小范围改动。这种坏味道通常表明代码的职责分散，导致</font>**<font style="color:rgb(64, 64, 64);">高耦合</font>**<font style="color:rgb(64, 64, 64);">和</font>**<font style="color:rgb(64, 64, 64);">低内聚</font>**<font style="color:rgb(64, 64, 64);">。与发散式变化（一个类因不同原因被频繁修改）不同，散弹式修改强调修改的“分散性”——同一功能逻辑分散在多个地方。</font>

---

### <font style="color:rgb(64, 64, 64);">示例场景</font>
<font style="color:rgb(64, 64, 64);">假设我们有一个订单处理系统，需要计算订单总价并记录日志。当前代码中，</font>**<font style="color:rgb(64, 64, 64);">折扣逻辑</font>**<font style="color:rgb(64, 64, 64);">（如固定折扣、百分比折扣）被硬编码在</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Order</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类和</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Logger</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类中。当新增一种折扣类型时，需要同时修改多个类。</font>

---

### <font style="color:rgb(64, 64, 64);">重构前代码</font>
<font style="color:rgb(255, 255, 255);background-color:rgb(80, 80, 90);">cpp</font>

<font style="background-color:rgb(80, 80, 90);">复制</font>

```plain
#include <iostream>
#include <string>

// 订单类（硬编码折扣逻辑）
class Order {
public:
    double calculateTotal(double price, const std::string& discountType) {
        double total = price;

        if (discountType == "Fixed10") {
            total -= 10; // 固定减10元
        } else if (discountType == "Percent20") {
            total *= 0.8; // 20% 折扣
        }

        return total;
    }
};

// 日志类（同样硬编码折扣逻辑）
class Logger {
public:
    void logDiscount(const std::string& discountType) {
        if (discountType == "Fixed10") {
            std::cout << "Applied Fixed10 discount." << std::endl;
        } else if (discountType == "Percent20") {
            std::cout << "Applied 20% discount." << std::endl;
        }
    }
};

// 客户端代码
int main() {
    Order order;
    Logger logger;

    double total = order.calculateTotal(100.0, "Percent20");
    logger.logDiscount("Percent20");
    std::cout << "Total price: " << total << std::endl; // 输出 80

    return 0;
}
```

---

### <font style="color:rgb(64, 64, 64);">问题分析</font>
1. **<font style="color:rgb(64, 64, 64);">散弹式修改</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - <font style="color:rgb(64, 64, 64);">新增一种折扣类型（如</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">"Fixed20"</font>`<font style="color:rgb(64, 64, 64);">）时，需要同时修改</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Order::calculateTotal</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">和</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Logger::logDiscount</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">中的条件判断逻辑。</font>
    - <font style="color:rgb(64, 64, 64);">同一功能（折扣逻辑）分散在多个类中，导致修改点分散。</font>
2. **<font style="color:rgb(64, 64, 64);">高耦合</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - `<font style="color:rgb(64, 64, 64);">Order</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">和</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Logger</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类直接依赖具体的折扣类型字符串，任何折扣逻辑的变化都会影响这两个类。</font>
3. **<font style="color:rgb(64, 64, 64);">重复代码</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - <font style="color:rgb(64, 64, 64);">折扣类型的判断逻辑在多个类中重复出现。</font>

---

### <font style="color:rgb(64, 64, 64);">重构思路</font>
1. **<font style="color:rgb(64, 64, 64);">集中折扣逻辑</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - <font style="color:rgb(64, 64, 64);">将折扣逻辑抽象为独立的策略类（Strategy Pattern），统一管理折扣计算。</font>
    - <font style="color:rgb(64, 64, 64);">使用工厂模式创建折扣对象。</font>
2. **<font style="color:rgb(64, 64, 64);">消除重复判断</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - <font style="color:rgb(64, 64, 64);">通过多态替代条件分支（Replace Conditional with Polymorphism）。</font>
3. **<font style="color:rgb(64, 64, 64);">统一日志接口</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - <font style="color:rgb(64, 64, 64);">将日志记录逻辑集成到折扣类中，避免分散的日志判断。</font>

---

### <font style="color:rgb(64, 64, 64);">重构后代码</font>
<font style="color:rgb(255, 255, 255);background-color:rgb(80, 80, 90);">cpp</font>

<font style="background-color:rgb(80, 80, 90);">复制</font>

```plain
#include <iostream>
#include <string>
#include <memory>
#include <unordered_map>

// 抽象折扣接口
class DiscountStrategy {
public:
    virtual double applyDiscount(double price) = 0;
    virtual std::string getDescription() const = 0;
    virtual ~DiscountStrategy() = default;
};

// 具体折扣：固定减10元
class Fixed10Discount : public DiscountStrategy {
public:
    double applyDiscount(double price) override {
        return price - 10;
    }

    std::string getDescription() const override {
        return "Fixed10 discount applied.";
    }
};

// 具体折扣：20% 折扣
class Percent20Discount : public DiscountStrategy {
public:
    double applyDiscount(double price) override {
        return price * 0.8;
    }

    std::string getDescription() const override {
        return "20% discount applied.";
    }
};

// 折扣工厂类
class DiscountFactory {
public:
    static std::unique_ptr<DiscountStrategy> createDiscount(const std::string& type) {
        if (type == "Fixed10") {
            return std::make_unique<Fixed10Discount>();
        } else if (type == "Percent20") {
            return std::make_unique<Percent20Discount>();
        }
        throw std::invalid_argument("Unknown discount type.");
    }
};

// 订单类（职责单一）
class Order {
public:
    double calculateTotal(double price, std::unique_ptr<DiscountStrategy> discount) {
        return discount->applyDiscount(price);
    }
};

// 日志类（直接使用折扣接口）
class Logger {
public:
    void logDiscount(const DiscountStrategy& discount) {
        std::cout << discount.getDescription() << std::endl;
    }
};

// 客户端代码
int main() {
    Order order;
    Logger logger;

    auto discount = DiscountFactory::createDiscount("Percent20");
    double total = order.calculateTotal(100.0, std::move(discount));
    logger.logDiscount(*discount);

    std::cout << "Total price: " << total << std::endl; // 输出 80

    return 0;
}
```

---

### <font style="color:rgb(64, 64, 64);">重构后优点</font>
1. **<font style="color:rgb(64, 64, 64);">消除散弹式修改</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - <font style="color:rgb(64, 64, 64);">新增折扣类型（如</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Fixed20</font>`<font style="color:rgb(64, 64, 64);">）时，只需添加新的</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">DiscountStrategy</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">子类并在工厂中注册，无需修改</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Order</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">或</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Logger</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类。</font>
2. **<font style="color:rgb(64, 64, 64);">职责集中</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - <font style="color:rgb(64, 64, 64);">折扣逻辑集中在</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">DiscountStrategy</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">及其子类中。</font>
    - <font style="color:rgb(64, 64, 64);">日志描述由折扣类自行提供，避免分散的判断。</font>
3. **<font style="color:rgb(64, 64, 64);">符合开闭原则</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - <font style="color:rgb(64, 64, 64);">扩展新功能（如折扣类型）时只需添加代码，无需修改已有代码。</font>

### <font style="color:rgb(64, 64, 64);">总结</font>
**<font style="color:rgb(64, 64, 64);">散弹式修改</font>**<font style="color:rgb(64, 64, 64);">的本质是相同功能的逻辑分散在多个地方，导致修改成本高且易出错。通过以下方法解决：</font>

1. **<font style="color:rgb(64, 64, 64);">抽象与封装</font>**<font style="color:rgb(64, 64, 64);">：将变化的部分（如折扣逻辑）抽象为接口。</font>
2. **<font style="color:rgb(64, 64, 64);">多态替代条件分支</font>**<font style="color:rgb(64, 64, 64);">：通过子类实现不同行为。</font>
3. **<font style="color:rgb(64, 64, 64);">集中管理</font>**<font style="color:rgb(64, 64, 64);">：使用工厂模式统一创建对象。</font>

<font style="color:rgb(64, 64, 64);">这种重构显著提高了代码的可维护性和可扩展性，是应对复杂业务逻辑变化的有效手段。</font>
