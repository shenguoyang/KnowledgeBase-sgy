#### <font style="color:rgb(64, 64, 64);">什么是“纯数据类”？</font>
**<font style="color:rgb(64, 64, 64);">纯数据类</font>**<font style="color:rgb(64, 64, 64);">指仅包含数据字段和简单getter/setter方法，</font>**<font style="color:rgb(64, 64, 64);">缺乏具体行为</font>**<font style="color:rgb(64, 64, 64);">的类。这类类本质上是数据的"哑容器"，常见问题包括：</font>

+ <font style="color:rgb(64, 64, 64);">数据验证逻辑散落在其他类中</font>
+ <font style="color:rgb(64, 64, 64);">违反"信息专家"原则（数据应与操作它的行为在一起）</font>
+ <font style="color:rgb(64, 64, 64);">容易导致贫血模型（Anemic Domain Model）</font>

---

### <font style="color:rgb(64, 64, 64);">问题示例：缺乏行为的哑数据容器</font>
<font style="color:rgb(255, 255, 255);background-color:rgb(80, 80, 90);">cpp</font>

<font style="background-color:rgb(80, 80, 90);">复制</font>

```plain
#include <string>

// 纯数据类：仅有数据字段和简单访问方法
class UserInfo {
private:
    std::string name;
    int age;
    std::string email;

public:
    // Getters/Setters
    void setName(const std::string& name) { this->name = name; }
    std::string getName() const { return name; }

    void setAge(int age) { this->age = age; }
    int getAge() const { return age; }

    void setEmail(const std::string& email) { this->email = email; }
    std::string getEmail() const { return email; }
};

// 数据验证逻辑散落在外部
bool validateUser(const UserInfo& user) {
    if (user.getName().empty()) return false;
    if (user.getAge() < 0) return false;
    if (user.getEmail().find('@') == std::string::npos) return false;
    return true;
}
```

#### <font style="color:rgb(64, 64, 64);">问题分析</font>
+ **<font style="color:rgb(64, 64, 64);">贫血模型</font>**<font style="color:rgb(64, 64, 64);">：所有业务逻辑都存在于</font>`<font style="color:rgb(64, 64, 64);">UserInfo</font>`<font style="color:rgb(64, 64, 64);">类外部</font>
+ **<font style="color:rgb(64, 64, 64);">数据脆弱性</font>**<font style="color:rgb(64, 64, 64);">：外部可直接修改数据导致无效状态</font>
+ **<font style="color:rgb(64, 64, 64);">重复验证</font>**<font style="color:rgb(64, 64, 64);">：多个地方需要复制粘贴</font>`<font style="color:rgb(64, 64, 64);">validateUser</font>`<font style="color:rgb(64, 64, 64);">逻辑</font>

### <font style="color:rgb(64, 64, 64);">重构方法：将行为内聚到数据类</font>
#### <font style="color:rgb(64, 64, 64);">重构方案1：内聚数据验证逻辑</font>
<font style="color:rgb(255, 255, 255);background-color:rgb(80, 80, 90);">cpp</font>

<font style="background-color:rgb(80, 80, 90);">复制</font>

```plain
#include <string>
#include <stdexcept>

class UserInfo {
private:
    std::string name;
    int age;
    std::string email;

    // 私有验证方法
    void validateName(const std::string& name) const {
        if (name.empty()) throw std::invalid_argument("Name cannot be empty");
    }

    void validateAge(int age) const {
        if (age < 0) throw std::invalid_argument("Age cannot be negative");
    }

    void validateEmail(const std::string& email) const {
        if (email.find('@') == std::string::npos) 
            throw std::invalid_argument("Invalid email format");
    }

public:
    // 构造函数集成验证
    UserInfo(const std::string& name, int age, const std::string& email) 
        : name(name), age(age), email(email) 
    {
        validateName(name);
        validateAge(age);
        validateEmail(email);
    }

    // 修改方法集成验证
    void setName(const std::string& newName) {
        validateName(newName);
        name = newName;
    }

    // 其他setter同理...
    
    // 业务方法示例
    bool isAdult() const { return age >= 18; }
    std::string getDisplayName() const { return name + " <" + email + ">"; }
};
```

#### <font style="color:rgb(64, 64, 64);">重构方案2：工厂方法创建有效对象</font>
<font style="color:rgb(255, 255, 255);background-color:rgb(80, 80, 90);">cpp</font>

<font style="background-color:rgb(80, 80, 90);">复制</font>

```plain
class UserInfo {
    // ...同上字段和验证方法...

public:
    // 工厂方法
    static UserInfo createValidated(
        const std::string& name,
        int age,
        const std::string& email
    ) {
        return UserInfo(name, age, email); // 调用私有构造函数
    }

private:
    // 强制通过工厂方法创建
    UserInfo(const std::string& name, int age, const std::string& email)
        : name(name), age(age), email(email) {}
};
```

---

### <font style="color:rgb(64, 64, 64);">重构后的优势</font>
1. **<font style="color:rgb(64, 64, 64);">自验证对象</font>**<font style="color:rgb(64, 64, 64);">：保证对象始终处于有效状态</font>
2. **<font style="color:rgb(64, 64, 64);">减少重复代码</font>**<font style="color:rgb(64, 64, 64);">：验证逻辑内聚在类内部</font>
3. **<font style="color:rgb(64, 64, 64);">丰富领域模型</font>**<font style="color:rgb(64, 64, 64);">：添加</font>`<font style="color:rgb(64, 64, 64);">isAdult()</font>`<font style="color:rgb(64, 64, 64);">等业务相关方法</font>
4. **<font style="color:rgb(64, 64, 64);">控制创建过程</font>**<font style="color:rgb(64, 64, 64);">：通过工厂方法确保对象有效性</font>

---

### <font style="color:rgb(64, 64, 64);">何时需要重构？</font>
+ <font style="color:rgb(64, 64, 64);">当类只有数据字段和简单getter/setter时</font>
+ <font style="color:rgb(64, 64, 64);">当发现多个类包含相同的数据操作逻辑时</font>
+ <font style="color:rgb(64, 64, 64);">当需要频繁检查数据有效性时</font>

### <font style="color:rgb(64, 64, 64);">高级技巧：值对象模式</font>
<font style="color:rgb(64, 64, 64);">对于不可变数据类，可将其设计为</font>**<font style="color:rgb(64, 64, 64);">值对象</font>**<font style="color:rgb(64, 64, 64);">（Value Object）：</font>

<font style="color:rgb(255, 255, 255);background-color:rgb(80, 80, 90);">cpp</font>

<font style="background-color:rgb(80, 80, 90);">复制</font>

```plain
class ImmutableUserInfo {
private:
    const std::string name;
    const int age;
    const std::string email;

public:
    ImmutableUserInfo(std::string name, int age, std::string email)
        : name(std::move(name)), age(age), email(std::move(email)) 
    {
        // 验证逻辑...
    }

    // 只提供访问方法，没有修改方法
    std::string getName() const { return name; }
    int getAge() const { return age; }
    std::string getEmail() const { return email; }
};
```

### <font style="color:rgb(64, 64, 64);">总结</font>
+ **<font style="color:rgb(64, 64, 64);">纯数据类</font>**<font style="color:rgb(64, 64, 64);">是典型的贫血模型表现</font>
+ **<font style="color:rgb(64, 64, 64);">重构核心</font>**<font style="color:rgb(64, 64, 64);">：将相关行为移入数据类，使其成为真正的领域对象</font>
+ **<font style="color:rgb(64, 64, 64);">关键原则</font>**<font style="color:rgb(64, 64, 64);">：遵守信息专家原则、保持高内聚低耦合</font>
+ **<font style="color:rgb(64, 64, 64);">延伸方向</font>**<font style="color:rgb(64, 64, 64);">：结合DDD（领域驱动设计）的价值对象、聚合根等概念</font>
