#### 什么是“纯数据类”？
**纯数据类**指仅包含数据字段和简单getter/setter方法，**缺乏具体行为**的类。这类类本质上是数据的"哑容器"，常见问题包括：

+ 数据验证逻辑散落在其他类中
+ 违反"信息专家"原则（数据应与操作它的行为在一起）
+ 容易导致贫血模型（Anemic Domain Model）

---

### 问题示例：缺乏行为的哑数据容器
cpp

复制

```plain
#include <string>

// 纯数据类：仅有数据字段和简单访问方法
class UserInfo {
private:
    std::string name;
    int age;
    std::string email;

public:
    // Getters/Setters
    void setName(const std::string& name) { this->name = name; }
    std::string getName() const { return name; }

    void setAge(int age) { this->age = age; }
    int getAge() const { return age; }

    void setEmail(const std::string& email) { this->email = email; }
    std::string getEmail() const { return email; }
};

// 数据验证逻辑散落在外部
bool validateUser(const UserInfo& user) {
    if (user.getName().empty()) return false;
    if (user.getAge() < 0) return false;
    if (user.getEmail().find('@') == std::string::npos) return false;
    return true;
}
```

#### 问题分析
+ **贫血模型**：所有业务逻辑都存在于`UserInfo`类外部
+ **数据脆弱性**：外部可直接修改数据导致无效状态
+ **重复验证**：多个地方需要复制粘贴`validateUser`逻辑

### 重构方法：将行为内聚到数据类
#### 重构方案1：内聚数据验证逻辑
cpp

复制

```plain
#include <string>
#include <stdexcept>

class UserInfo {
private:
    std::string name;
    int age;
    std::string email;

    // 私有验证方法
    void validateName(const std::string& name) const {
        if (name.empty()) throw std::invalid_argument("Name cannot be empty");
    }

    void validateAge(int age) const {
        if (age < 0) throw std::invalid_argument("Age cannot be negative");
    }

    void validateEmail(const std::string& email) const {
        if (email.find('@') == std::string::npos) 
            throw std::invalid_argument("Invalid email format");
    }

public:
    // 构造函数集成验证
    UserInfo(const std::string& name, int age, const std::string& email) 
        : name(name), age(age), email(email) 
    {
        validateName(name);
        validateAge(age);
        validateEmail(email);
    }

    // 修改方法集成验证
    void setName(const std::string& newName) {
        validateName(newName);
        name = newName;
    }

    // 其他setter同理...
    
    // 业务方法示例
    bool isAdult() const { return age >= 18; }
    std::string getDisplayName() const { return name + " <" + email + ">"; }
};
```

#### 重构方案2：工厂方法创建有效对象
cpp

复制

```plain
class UserInfo {
    // ...同上字段和验证方法...

public:
    // 工厂方法
    static UserInfo createValidated(
        const std::string& name,
        int age,
        const std::string& email
    ) {
        return UserInfo(name, age, email); // 调用私有构造函数
    }

private:
    // 强制通过工厂方法创建
    UserInfo(const std::string& name, int age, const std::string& email)
        : name(name), age(age), email(email) {}
};
```

---

### 重构后的优势
1. **自验证对象**：保证对象始终处于有效状态
2. **减少重复代码**：验证逻辑内聚在类内部
3. **丰富领域模型**：添加`isAdult()`等业务相关方法
4. **控制创建过程**：通过工厂方法确保对象有效性

---

### 何时需要重构？
+ 当类只有数据字段和简单getter/setter时
+ 当发现多个类包含相同的数据操作逻辑时
+ 当需要频繁检查数据有效性时

### 高级技巧：值对象模式
对于不可变数据类，可将其设计为**值对象**（Value Object）：

cpp

复制

```plain
class ImmutableUserInfo {
private:
    const std::string name;
    const int age;
    const std::string email;

public:
    ImmutableUserInfo(std::string name, int age, std::string email)
        : name(std::move(name)), age(age), email(std::move(email)) 
    {
        // 验证逻辑...
    }

    // 只提供访问方法，没有修改方法
    std::string getName() const { return name; }
    int getAge() const { return age; }
    std::string getEmail() const { return email; }
};
```

---

### 总结
+ **纯数据类**是典型的贫血模型表现
+ **重构核心**：将相关行为移入数据类，使其成为真正的领域对象
+ **关键原则**：遵守信息专家原则、保持高内聚低耦合
+ **延伸方向**：结合DDD（领域驱动设计）的价值对象、聚合根等概念
