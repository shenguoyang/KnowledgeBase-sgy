### <font style="color:rgb(64, 64, 64);">代码坏味道：</font>**<font style="color:rgb(64, 64, 64);">发散式变化（Divergent Change）</font>**
#### <font style="color:rgb(64, 64, 64);">定义</font>
<font style="color:rgb(64, 64, 64);">发散式变化是指一个类因为</font>**<font style="color:rgb(64, 64, 64);">不同的原因</font>**<font style="color:rgb(64, 64, 64);">而需要频繁修改。换句话说，当需求变化时，你需要在同一个类的多个地方进行修改，而不是集中在一个地方。这种坏味道通常表明类的职责不够单一，违反了</font>**<font style="color:rgb(64, 64, 64);">单一职责原则（SRP）</font>**<font style="color:rgb(64, 64, 64);">。</font>

#### <font style="color:rgb(64, 64, 64);">示例场景</font>
<font style="color:rgb(64, 64, 64);">假设我们有一个</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Report</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类，它负责生成报告并将其保存为不同的格式（如</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">PDF</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">和</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">CSV</font>`<font style="color:rgb(64, 64, 64);">）。随着需求的变化，我们可能需要修改报告的生成逻辑或保存逻辑，而这些修改通常发生在同一个类中。</font>

---

### <font style="color:rgb(64, 64, 64);">重构前代码</font>
<font style="color:rgb(255, 255, 255);background-color:rgb(80, 80, 90);">cpp</font>

<font style="background-color:rgb(80, 80, 90);">复制</font>

```plain
#include <iostream>
#include <string>

// 报告类（违反单一职责原则）
class Report {
public:
    void generateReport() {
        // 生成报告的逻辑
        std::cout << "Generating report content..." << std::endl;
    }

    void saveToPdf(const std::string& filePath) {
        // 保存为 PDF 的逻辑
        std::cout << "Saving report to PDF: " << filePath << std::endl;
    }

    void saveToCsv(const std::string& filePath) {
        // 保存为 CSV 的逻辑
        std::cout << "Saving report to CSV: " << filePath << std::endl;
    }
};

// 客户端代码
int main() {
    Report report;
    report.generateReport();
    report.saveToPdf("report.pdf");
    report.saveToCsv("report.csv");

    return 0;
}
```

---

### <font style="color:rgb(64, 64, 64);">问题分析</font>
1. **<font style="color:rgb(64, 64, 64);">发散式变化</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - <font style="color:rgb(64, 64, 64);">如果报告生成逻辑需要修改（例如添加新的数据字段），我们需要修改</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">generateReport()</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">方法。</font>
    - <font style="color:rgb(64, 64, 64);">如果保存逻辑需要修改（例如支持新的格式），我们需要修改</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">saveToPdf()</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">或</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">saveToCsv()</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">方法。</font>
    - <font style="color:rgb(64, 64, 64);">这些修改的原因不同，但都集中在同一个类中，导致类不稳定。</font>
2. **<font style="color:rgb(64, 64, 64);">违反单一职责原则</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - `<font style="color:rgb(64, 64, 64);">Report</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类同时负责生成报告和保存报告，职责过多。</font>

---

### <font style="color:rgb(64, 64, 64);">重构思路</font>
1. **<font style="color:rgb(64, 64, 64);">分离职责</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - <font style="color:rgb(64, 64, 64);">将报告生成和保存的逻辑分离到不同的类中。</font>
    - <font style="color:rgb(64, 64, 64);">使用策略模式（Strategy Pattern）来封装不同的保存逻辑。</font>
2. **<font style="color:rgb(64, 64, 64);">引入接口</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - <font style="color:rgb(64, 64, 64);">定义一个</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">ReportSaver</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">接口，用于统一保存行为。</font>
    - <font style="color:rgb(64, 64, 64);">具体的保存逻辑由实现类（如</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">PdfSaver</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">和</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">CsvSaver</font>`<font style="color:rgb(64, 64, 64);">）完成。</font>

---

### <font style="color:rgb(64, 64, 64);">重构后代码</font>
<font style="color:rgb(255, 255, 255);background-color:rgb(80, 80, 90);">cpp</font>

<font style="background-color:rgb(80, 80, 90);">复制</font>

```plain
#include <iostream>
#include <string>
#include <memory>

// 报告保存接口
class ReportSaver {
public:
    virtual void save(const std::string& filePath) = 0;
    virtual ~ReportSaver() = default;
};

// 具体保存类：PDF
class PdfSaver : public ReportSaver {
public:
    void save(const std::string& filePath) override {
        std::cout << "Saving report to PDF: " << filePath << std::endl;
    }
};

// 具体保存类：CSV
class CsvSaver : public ReportSaver {
public:
    void save(const std::string& filePath) override {
        std::cout << "Saving report to CSV: " << filePath << std::endl;
    }
};

// 报告类（职责单一）
class Report {
public:
    void generateReport() {
        // 生成报告的逻辑
        std::cout << "Generating report content..." << std::endl;
    }

    void saveReport(std::unique_ptr<ReportSaver> saver, const std::string& filePath) {
        saver->save(filePath);
    }
};

// 客户端代码
int main() {
    Report report;
    report.generateReport();

    // 保存为 PDF
    report.saveReport(std::make_unique<PdfSaver>(), "report.pdf");

    // 保存为 CSV
    report.saveReport(std::make_unique<CsvSaver>(), "report.csv");

    return 0;
}
```

---

### <font style="color:rgb(64, 64, 64);">重构后优点</font>
1. **<font style="color:rgb(64, 64, 64);">职责分离</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - `<font style="color:rgb(64, 64, 64);">Report</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类只负责生成报告，保存逻辑由</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">ReportSaver</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">及其子类负责。</font>
    - <font style="color:rgb(64, 64, 64);">符合单一职责原则，类的职责更加清晰。</font>
2. **<font style="color:rgb(64, 64, 64);">扩展性增强</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - <font style="color:rgb(64, 64, 64);">如果需要支持新的保存格式（如</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Excel</font>`<font style="color:rgb(64, 64, 64);">），只需新增一个</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">ExcelSaver</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类，无需修改</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Report</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类。</font>
    - <font style="color:rgb(64, 64, 64);">符合开闭原则（对扩展开放，对修改关闭）。</font>
3. **<font style="color:rgb(64, 64, 64);">减少发散式变化</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - <font style="color:rgb(64, 64, 64);">修改报告生成逻辑时，只需修改</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Report</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类。</font>
    - <font style="color:rgb(64, 64, 64);">修改保存逻辑时，只需修改具体的</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">ReportSaver</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">实现类。</font>

### <font style="color:rgb(64, 64, 64);">总结</font>
<font style="color:rgb(64, 64, 64);">通过将 </font>**<font style="color:rgb(64, 64, 64);">发散式变化</font>**<font style="color:rgb(64, 64, 64);"> 的代码重构为职责单一、结构清晰的类，我们不仅解决了代码坏味道，还提高了代码的可维护性和扩展性。这种重构方法在大型项目中尤为重要，能够有效降低代码的复杂性和耦合度。</font>
