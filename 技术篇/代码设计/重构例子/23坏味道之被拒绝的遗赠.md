#### 什么是“被拒绝的遗赠”？
当子类继承了父类的数据或方法，但未使用或拒绝使用部分继承内容时，就出现了**被拒绝的遗赠**。这种坏味道表明继承关系可能不恰当，违反了“is-a”原则。常见表现包括：

+ 子类重写父类方法并抛出异常。
+ 子类仅使用父类的部分功能，其余功能被闲置。

#### 问题示例：错误的继承关系
假设有一个 `Bird` 基类，提供 `fly()` 和 `eat()` 方法。企鹅（`Penguin`）继承自 `Bird`，但企鹅并不会飞。为了处理这种矛盾，子类被迫重写 `fly()` 方法并抛出异常。

cpp

复制

```plain
#include <stdexcept>

// 基类：鸟
class Bird {
public:
    virtual void fly() {
        // 默认飞行逻辑
    }
    virtual void eat() {
        // 进食逻辑
    }
};

// 子类：企鹅（不会飞）
class Penguin : public Bird {
public:
    void fly() override {
        throw std::runtime_error("Penguins can't fly!");
    }
};
```

#### 问题分析
+ **违反里氏替换原则**：子类无法无缝替换父类，调用 `Penguin` 的 `fly()` 会抛出异常。
+ **继承关系不合理**：企鹅是鸟，但不会飞，直接继承 `Bird` 会导致接口污染。

### 重构方法：分离接口与组合代替继承
#### 重构方案：拆分层次结构
1. **提取飞行能力为独立接口**，仅让会飞的鸟类实现它。
2. **重构继承链**，确保子类不需要拒绝父类的功能。

cpp

复制

```plain
// 基类：鸟（仅保留通用功能）
class Bird {
public:
    virtual void eat() = 0;
    virtual ~Bird() = default;
};

// 飞行能力接口
class Flyable {
public:
    virtual void fly() = 0;
    virtual ~Flyable() = default;
};

// 会飞的鸟类：继承Bird并实现Flyable
class Sparrow : public Bird, public Flyable {
public:
    void eat() override {
        // 麻雀进食逻辑
    }
    void fly() override {
        // 麻雀飞行逻辑
    }
};

// 企鹅：继承Bird但不实现Flyable
class Penguin : public Bird {
public:
    void eat() override {
        // 企鹅进食逻辑
    }
};
```

#### 重构后的优势
+ **消除被拒绝的遗赠**：`Penguin` 不再需要处理 `fly()`。
+ **符合接口隔离原则**：类只实现与自身相关的方法。
+ **增强扩展性**：新增不会飞的鸟类时，无需处理无关接口。

### 何时使用组合代替继承？
如果子类仅需复用父类的部分代码，且与父类没有严格的“is-a”关系，则**组合**更合适。例如：

cpp

复制

```plain
// 飞行能力实现类
class FlightCapability {
public:
    void fly() {
        // 飞行逻辑
    }
};

// 通过组合复用飞行能力
class Sparrow : public Bird {
private:
    FlightCapability flier;
public:
    void eat() override { /* ... */ }
    void fly() { flier.fly(); } // 委托给组合对象
};
```

---

### 总结
+ **被拒绝的遗赠**表明继承层次设计不合理。
+ **重构关键**：通过拆分接口、组合或调整继承链，确保子类无需拒绝父类功能。
+ **核心原则**：遵循里氏替换原则（LSP）和接口隔离原则（ISP）。
