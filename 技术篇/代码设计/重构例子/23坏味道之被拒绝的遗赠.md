#### <font style="color:rgb(64, 64, 64);">什么是“被拒绝的遗赠”？</font>
<font style="color:rgb(64, 64, 64);">当子类继承了父类的数据或方法，但未使用或拒绝使用部分继承内容时，就出现了</font>**<font style="color:rgb(64, 64, 64);">被拒绝的遗赠</font>**<font style="color:rgb(64, 64, 64);">。这种坏味道表明继承关系可能不恰当，违反了“is-a”原则。常见表现包括：</font>

+ <font style="color:rgb(64, 64, 64);">子类重写父类方法并抛出异常。</font>
+ <font style="color:rgb(64, 64, 64);">子类仅使用父类的部分功能，其余功能被闲置。</font>

#### <font style="color:rgb(64, 64, 64);">问题示例：错误的继承关系</font>
<font style="color:rgb(64, 64, 64);">假设有一个</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Bird</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">基类，提供</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">fly()</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">和</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">eat()</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">方法。企鹅（</font>`<font style="color:rgb(64, 64, 64);">Penguin</font>`<font style="color:rgb(64, 64, 64);">）继承自</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Bird</font>`<font style="color:rgb(64, 64, 64);">，但企鹅并不会飞。为了处理这种矛盾，子类被迫重写</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">fly()</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">方法并抛出异常。</font>

<font style="color:rgb(255, 255, 255);background-color:rgb(80, 80, 90);">cpp</font>

<font style="background-color:rgb(80, 80, 90);">复制</font>

```plain
#include <stdexcept>

// 基类：鸟
class Bird {
public:
    virtual void fly() {
        // 默认飞行逻辑
    }
    virtual void eat() {
        // 进食逻辑
    }
};

// 子类：企鹅（不会飞）
class Penguin : public Bird {
public:
    void fly() override {
        throw std::runtime_error("Penguins can't fly!");
    }
};
```

#### <font style="color:rgb(64, 64, 64);">问题分析</font>
+ **<font style="color:rgb(64, 64, 64);">违反里氏替换原则</font>**<font style="color:rgb(64, 64, 64);">：子类无法无缝替换父类，调用</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Penguin</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">的</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">fly()</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">会抛出异常。</font>
+ **<font style="color:rgb(64, 64, 64);">继承关系不合理</font>**<font style="color:rgb(64, 64, 64);">：企鹅是鸟，但不会飞，直接继承</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Bird</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">会导致接口污染。</font>

---

### <font style="color:rgb(64, 64, 64);">重构方法：分离接口与组合代替继承</font>
#### <font style="color:rgb(64, 64, 64);">重构方案：拆分层次结构</font>
1. **<font style="color:rgb(64, 64, 64);">提取飞行能力为独立接口</font>**<font style="color:rgb(64, 64, 64);">，仅让会飞的鸟类实现它。</font>
2. **<font style="color:rgb(64, 64, 64);">重构继承链</font>**<font style="color:rgb(64, 64, 64);">，确保子类不需要拒绝父类的功能。</font>

<font style="color:rgb(255, 255, 255);background-color:rgb(80, 80, 90);">cpp</font>

<font style="background-color:rgb(80, 80, 90);">复制</font>

```plain
// 基类：鸟（仅保留通用功能）
class Bird {
public:
    virtual void eat() = 0;
    virtual ~Bird() = default;
};

// 飞行能力接口
class Flyable {
public:
    virtual void fly() = 0;
    virtual ~Flyable() = default;
};

// 会飞的鸟类：继承Bird并实现Flyable
class Sparrow : public Bird, public Flyable {
public:
    void eat() override {
        // 麻雀进食逻辑
    }
    void fly() override {
        // 麻雀飞行逻辑
    }
};

// 企鹅：继承Bird但不实现Flyable
class Penguin : public Bird {
public:
    void eat() override {
        // 企鹅进食逻辑
    }
};
```

#### <font style="color:rgb(64, 64, 64);">重构后的优势</font>
+ **<font style="color:rgb(64, 64, 64);">消除被拒绝的遗赠</font>**<font style="color:rgb(64, 64, 64);">：</font>`<font style="color:rgb(64, 64, 64);">Penguin</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">不再需要处理</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">fly()</font>`<font style="color:rgb(64, 64, 64);">。</font>
+ **<font style="color:rgb(64, 64, 64);">符合接口隔离原则</font>**<font style="color:rgb(64, 64, 64);">：类只实现与自身相关的方法。</font>
+ **<font style="color:rgb(64, 64, 64);">增强扩展性</font>**<font style="color:rgb(64, 64, 64);">：新增不会飞的鸟类时，无需处理无关接口。</font>

---

### <font style="color:rgb(64, 64, 64);">何时使用组合代替继承？</font>
<font style="color:rgb(64, 64, 64);">如果子类仅需复用父类的部分代码，且与父类没有严格的“is-a”关系，则</font>**<font style="color:rgb(64, 64, 64);">组合</font>**<font style="color:rgb(64, 64, 64);">更合适。例如：</font>

<font style="color:rgb(255, 255, 255);background-color:rgb(80, 80, 90);">cpp</font>

<font style="background-color:rgb(80, 80, 90);">复制</font>

```plain
// 飞行能力实现类
class FlightCapability {
public:
    void fly() {
        // 飞行逻辑
    }
};

// 通过组合复用飞行能力
class Sparrow : public Bird {
private:
    FlightCapability flier;
public:
    void eat() override { /* ... */ }
    void fly() { flier.fly(); } // 委托给组合对象
};
```

---

### <font style="color:rgb(64, 64, 64);">总结</font>
+ **<font style="color:rgb(64, 64, 64);">被拒绝的遗赠</font>**<font style="color:rgb(64, 64, 64);">表明继承层次设计不合理。</font>
+ **<font style="color:rgb(64, 64, 64);">重构关键</font>**<font style="color:rgb(64, 64, 64);">：通过拆分接口、组合或调整继承链，确保子类无需拒绝父类功能。</font>
+ **<font style="color:rgb(64, 64, 64);">核心原则</font>**<font style="color:rgb(64, 64, 64);">：遵循里氏替换原则（LSP）和接口隔离原则（ISP）。</font>
