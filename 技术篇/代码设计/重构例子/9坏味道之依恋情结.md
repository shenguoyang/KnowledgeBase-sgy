### <font style="color:rgb(64, 64, 64);">代码坏味道：</font>**<font style="color:rgb(64, 64, 64);">依恋情结（Feature Envy）</font>**
#### <font style="color:rgb(64, 64, 64);">定义</font>
**<font style="color:rgb(64, 64, 64);">依恋情结</font>**<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">是指一个类的方法过度依赖另一个类的数据（频繁访问或操作其他类的成员变量），而不是专注于自己的数据。这种现象表明该方法可能更适合放在被访问的类中，因为它的逻辑更贴近那个类的职责。这种坏味道违反了面向对象设计的</font>**<font style="color:rgb(64, 64, 64);">高内聚、低耦合</font>**<font style="color:rgb(64, 64, 64);">原则。</font>

---

### <font style="color:rgb(64, 64, 64);">示例场景</font>
<font style="color:rgb(64, 64, 64);">假设有一个</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Order</font>`<font style="color:rgb(64, 64, 64);">（订单）类和一个</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Customer</font>`<font style="color:rgb(64, 64, 64);">（客户）类。</font>`<font style="color:rgb(64, 64, 64);">Order</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类中有一个方法</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">calculateDiscount()</font>`<font style="color:rgb(64, 64, 64);">，该方法需要频繁访问</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Customer</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类的属性（如会员等级、历史消费金额）来计算折扣。显然，折扣逻辑更应属于</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Customer</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类的职责。</font>

---

### <font style="color:rgb(64, 64, 64);">重构前代码</font>
<font style="color:rgb(255, 255, 255);background-color:rgb(80, 80, 90);">cpp</font>

<font style="background-color:rgb(80, 80, 90);">复制</font>

```plain
#include <iostream>
#include <string>

class Customer {
public:
    Customer(int membershipLevel, double totalSpent)
        : membershipLevel_(membershipLevel), totalSpent_(totalSpent) {}

    int getMembershipLevel() const { return membershipLevel_; }
    double getTotalSpent() const { return totalSpent_; }

private:
    int membershipLevel_; // 会员等级（1-5）
    double totalSpent_;   // 历史消费总额
};

class Order {
public:
    Order(double amount, const Customer& customer)
        : amount_(amount), customer_(customer) {}

    // 问题方法：依恋情结
    double calculateDiscount() {
        double discount = 0.0;

        // 基于客户会员等级计算折扣
        if (customer_.getMembershipLevel() >= 3) {
            discount += 0.1; // 高级会员额外 10% 折扣
        }

        // 基于客户历史消费金额计算折扣
        if (customer_.getTotalSpent() > 10000) {
            discount += 0.05; // 大客户额外 5% 折扣
        }

        return amount_ * discount;
    }

private:
    double amount_;     // 订单金额
    Customer customer_; // 关联的客户
};

// 客户端代码
int main() {
    Customer customer(4, 15000.0);
    Order order(1000.0, customer);
    std::cout << "Discount: " << order.calculateDiscount() << std::endl; // 输出 150 (1000 * 0.15)

    return 0;
}
```

---

### <font style="color:rgb(64, 64, 64);">问题分析</font>
1. **<font style="color:rgb(64, 64, 64);">依恋情结</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - `<font style="color:rgb(64, 64, 64);">Order::calculateDiscount()</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">方法频繁访问</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Customer</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类的</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">getMembershipLevel()</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">和</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">getTotalSpent()</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">方法。</font>
    - <font style="color:rgb(64, 64, 64);">折扣逻辑实际上与客户属性强相关，应该由</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Customer</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类负责计算。</font>
2. **<font style="color:rgb(64, 64, 64);">职责错位</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - `<font style="color:rgb(64, 64, 64);">Order</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类本应关注订单本身的逻辑（如金额、状态），而折扣计算逻辑却耦合了客户属性。</font>
3. **<font style="color:rgb(64, 64, 64);">可维护性差</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - <font style="color:rgb(64, 64, 64);">如果客户折扣规则变化（例如新增会员等级），需要修改</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Order</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类，而非</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Customer</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类。</font>

---

### <font style="color:rgb(64, 64, 64);">重构思路</font>
1. **<font style="color:rgb(64, 64, 64);">移动方法</font>**<font style="color:rgb(64, 64, 64);">（Move Method）：</font>
    - <font style="color:rgb(64, 64, 64);">将</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">calculateDiscount()</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">方法移动到</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Customer</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类中，使其直接操作自己的数据。</font>
2. **<font style="color:rgb(64, 64, 64);">封装计算逻辑</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - `<font style="color:rgb(64, 64, 64);">Customer</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类提供</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">calculateDiscount(double orderAmount)</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">方法，根据自身属性计算折扣。</font>

---

### <font style="color:rgb(64, 64, 64);">重构后代码</font>
<font style="color:rgb(255, 255, 255);background-color:rgb(80, 80, 90);">cpp</font>

<font style="background-color:rgb(80, 80, 90);">复制</font>

```plain
#include <iostream>
#include <string>

class Customer {
public:
    Customer(int membershipLevel, double totalSpent)
        : membershipLevel_(membershipLevel), totalSpent_(totalSpent) {}

    // 将折扣计算逻辑移动到 Customer 类中
    double calculateDiscount(double orderAmount) const {
        double discount = 0.0;

        if (membershipLevel_ >= 3) {
            discount += 0.1;
        }

        if (totalSpent_ > 10000) {
            discount += 0.05;
        }

        return orderAmount * discount;
    }

private:
    int membershipLevel_;
    double totalSpent_;
};

class Order {
public:
    Order(double amount, const Customer& customer)
        : amount_(amount), customer_(customer) {}

    // 简化后的方法：直接委托给 Customer 类
    double getDiscount() const {
        return customer_.calculateDiscount(amount_);
    }

private:
    double amount_;
    Customer customer_;
};

// 客户端代码
int main() {
    Customer customer(4, 15000.0);
    Order order(1000.0, customer);
    std::cout << "Discount: " << order.getDiscount() << std::endl; // 输出 150

    return 0;
}
```

---

### <font style="color:rgb(64, 64, 64);">重构后优点</font>
1. **<font style="color:rgb(64, 64, 64);">职责清晰</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - `<font style="color:rgb(64, 64, 64);">Customer</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类负责管理客户属性及其相关的折扣逻辑，符合单一职责原则。</font>
    - `<font style="color:rgb(64, 64, 64);">Order</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类只需关注订单金额和获取折扣，不再耦合客户逻辑。</font>
2. **<font style="color:rgb(64, 64, 64);">消除依恋情结</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - `<font style="color:rgb(64, 64, 64);">Order::getDiscount()</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">方法不再直接操作</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Customer</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">的细节，而是通过调用其接口完成计算。</font>
3. **<font style="color:rgb(64, 64, 64);">扩展性增强</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - <font style="color:rgb(64, 64, 64);">新增折扣规则（例如根据客户地域调整折扣）时，只需修改</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Customer</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类，无需改动</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">Order</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">类。</font>

### <font style="color:rgb(64, 64, 64);">总结</font>
**<font style="color:rgb(64, 64, 64);">依恋情结</font>**<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">的根源是方法的逻辑与数据归属错位。通过重构：</font>

1. **<font style="color:rgb(64, 64, 64);">移动方法</font>**<font style="color:rgb(64, 64, 64);">：将逻辑移动到数据所在的类。</font>
2. **<font style="color:rgb(64, 64, 64);">封装行为</font>**<font style="color:rgb(64, 64, 64);">：让数据所有者提供相关操作接口。</font>

<font style="color:rgb(64, 64, 64);">这种重构不仅解决了代码坏味道，还提高了代码的内聚性和可维护性，是面向对象设计中“数据与行为封装在一起”原则的典型应用。</font>
