#### <font style="color:rgb(64, 64, 64);">什么是“异曲同工的类”？</font>
**<font style="color:rgb(64, 64, 64);">异曲同工的类</font>**<font style="color:rgb(64, 64, 64);">（或称为重复类）指多个类在功能、结构或实现上存在高度相似性，但未通过合理的抽象消除重复代码。这种代码坏味道会导致：</font>

+ **<font style="color:rgb(64, 64, 64);">维护困难</font>**<font style="color:rgb(64, 64, 64);">：修改逻辑时需同步修改多个类。</font>
+ **<font style="color:rgb(64, 64, 64);">冗余代码</font>**<font style="color:rgb(64, 64, 64);">：相同逻辑重复实现，违反 DRY（Don’t Repeat Yourself）原则。</font>
+ **<font style="color:rgb(64, 64, 64);">扩展性差</font>**<font style="color:rgb(64, 64, 64);">：新增类似功能时需要复制现有类。</font>

---

### <font style="color:rgb(64, 64, 64);">问题示例：重复的类结构</font>
<font style="color:rgb(64, 64, 64);">假设系统需要处理两种数据格式（</font>`<font style="color:rgb(64, 64, 64);">XML</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">和</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">JSON</font>`<font style="color:rgb(64, 64, 64);">），但设计了两个几乎相同的解析类，仅部分细节不同。</font>

<font style="color:rgb(255, 255, 255);background-color:rgb(80, 80, 90);">cpp</font>

<font style="background-color:rgb(80, 80, 90);">复制</font>

```plain
#include <string>
#include <iostream>

// XML 解析类
class XmlParser {
public:
    void parse(const std::string& data) {
        std::cout << "Parsing XML data: " << data << std::endl;
        // XML 解析逻辑...
        validateXml(data); // XML 特有验证
    }

private:
    void validateXml(const std::string& data) {
        if (data.find("<xml>") == std::string::npos) {
            throw std::invalid_argument("Invalid XML format");
        }
    }
};

// JSON 解析类（与 XmlParser 高度重复）
class JsonParser {
public:
    void parse(const std::string& data) {
        std::cout << "Parsing JSON data: " << data << std::endl;
        // JSON 解析逻辑...
        validateJson(data); // JSON 特有验证
    }

private:
    void validateJson(const std::string& data) {
        if (data.find("{") == std::string::npos) {
            throw std::invalid_argument("Invalid JSON format");
        }
    }
};

// 使用示例
int main() {
    XmlParser xmlParser;
    xmlParser.parse("<xml>...</xml>");

    JsonParser jsonParser;
    jsonParser.parse(R"({"key": "value"})");

    return 0;
}
```

#### <font style="color:rgb(64, 64, 64);">问题分析</font>
+ **<font style="color:rgb(64, 64, 64);">重复的接口</font>**<font style="color:rgb(64, 64, 64);">：</font>`<font style="color:rgb(64, 64, 64);">XmlParser</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">和</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">JsonParser</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">都有</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">parse()</font>`<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">方法，且流程相似。</font>
+ **<font style="color:rgb(64, 64, 64);">重复的通用逻辑</font>**<font style="color:rgb(64, 64, 64);">：如日志输出、异常处理可能重复。</font>
+ **<font style="color:rgb(64, 64, 64);">差异点未隔离</font>**<font style="color:rgb(64, 64, 64);">：仅验证逻辑不同，但未通过抽象统一处理。</font>

---

### <font style="color:rgb(64, 64, 64);">重构方法：抽象公共逻辑，隔离差异</font>
#### <font style="color:rgb(64, 64, 64);">重构方案1：提取基类 + 模板方法模式</font>
<font style="color:rgb(64, 64, 64);">将公共逻辑提取到基类中，并通过</font>**<font style="color:rgb(64, 64, 64);">虚函数</font>**<font style="color:rgb(64, 64, 64);">隔离差异点。</font>

<font style="color:rgb(255, 255, 255);background-color:rgb(80, 80, 90);">cpp</font>

<font style="background-color:rgb(80, 80, 90);">复制</font>

```plain
#include <string>
#include <iostream>
#include <stdexcept>

// 抽象基类：定义通用解析流程
class DataParser {
public:
    // 模板方法：定义解析流程框架
    void parse(const std::string& data) {
        std::cout << "Parsing data: " << data << std::endl;
        validate(data);  // 调用子类实现的验证逻辑
        process(data);   // 调用子类实现的具体处理逻辑
    }

    virtual ~DataParser() = default;

protected:
    // 纯虚函数：强制子类实现差异逻辑
    virtual void validate(const std::string& data) const = 0;
    virtual void process(const std::string& data) const = 0;
};

// XML 解析子类
class XmlParser : public DataParser {
protected:
    void validate(const std::string& data) const override {
        if (data.find("<xml>") == std::string::npos) {
            throw std::invalid_argument("Invalid XML format");
        }
    }

    void process(const std::string& data) const override {
        // XML 特有的处理逻辑...
        std::cout << "Processing XML data..." << std::endl;
    }
};

// JSON 解析子类
class JsonParser : public DataParser {
protected:
    void validate(const std::string& data) const override {
        if (data.find("{") == std::string::npos) {
            throw std::invalid_argument("Invalid JSON format");
        }
    }

    void process(const std::string& data) const override {
        // JSON 特有的处理逻辑...
        std::cout << "Processing JSON data..." << std::endl;
    }
};

// 使用示例
int main() {
    XmlParser xmlParser;
    xmlParser.parse("<xml>...</xml>");

    JsonParser jsonParser;
    jsonParser.parse(R"({"key": "value"})");

    return 0;
}
```

#### <font style="color:rgb(64, 64, 64);">重构方案2：策略模式（依赖注入）</font>
<font style="color:rgb(64, 64, 64);">将解析逻辑抽象为接口，通过组合实现灵活替换。</font>

<font style="color:rgb(255, 255, 255);background-color:rgb(80, 80, 90);">cpp</font>

<font style="background-color:rgb(80, 80, 90);">复制</font>

```plain
#include <string>
#include <iostream>
#include <memory>

// 解析策略接口
class ParseStrategy {
public:
    virtual void validate(const std::string& data) const = 0;
    virtual void process(const std::string& data) const = 0;
    virtual ~ParseStrategy() = default;
};

// 通用解析器（依赖策略接口）
class DataParser {
private:
    std::unique_ptr<ParseStrategy> strategy;

public:
    DataParser(std::unique_ptr<ParseStrategy> strat) 
        : strategy(std::move(strat)) {}

    void parse(const std::string& data) {
        std::cout << "Parsing data: " << data << std::endl;
        strategy->validate(data);
        strategy->process(data);
    }
};

// XML 策略实现
class XmlStrategy : public ParseStrategy {
public:
    void validate(const std::string& data) const override {
        if (data.find("<xml>") == std::string::npos) {
            throw std::invalid_argument("Invalid XML format");
        }
    }

    void process(const std::string& data) const override {
        std::cout << "Processing XML data..." << std::endl;
    }
};

// JSON 策略实现
class JsonStrategy : public ParseStrategy {
public:
    void validate(const std::string& data) const override {
        if (data.find("{") == std::string::npos) {
            throw std::invalid_argument("Invalid JSON format");
        }
    }

    void process(const std::string& data) const override {
        std::cout << "Processing JSON data..." << std::endl;
    }
};

// 使用示例
int main() {
    DataParser xmlParser(std::make_unique<XmlStrategy>());
    xmlParser.parse("<xml>...</xml>");

    DataParser jsonParser(std::make_unique<JsonStrategy>());
    jsonParser.parse(R"({"key": "value"})");

    return 0;
}
```

### <font style="color:rgb(64, 64, 64);">重构后的优势</font>
1. **<font style="color:rgb(64, 64, 64);">消除重复代码</font>**<font style="color:rgb(64, 64, 64);">：公共逻辑集中到基类或组合类中。</font>
2. **<font style="color:rgb(64, 64, 64);">提高扩展性</font>**<font style="color:rgb(64, 64, 64);">：新增格式（如</font><font style="color:rgb(64, 64, 64);"> </font>`<font style="color:rgb(64, 64, 64);">YAML</font>`<font style="color:rgb(64, 64, 64);">）只需添加新子类或策略。</font>
3. **<font style="color:rgb(64, 64, 64);">隔离变化点</font>**<font style="color:rgb(64, 64, 64);">：差异逻辑通过虚函数或接口实现，修改不影响整体流程。</font>
4. **<font style="color:rgb(64, 64, 64);">符合开闭原则</font>**<font style="color:rgb(64, 64, 64);">：对扩展开放，对修改关闭。</font>

---

### <font style="color:rgb(64, 64, 64);">何时需要重构？</font>
+ <font style="color:rgb(64, 64, 64);">当发现多个类实现相似的业务流程时。</font>
+ <font style="color:rgb(64, 64, 64);">当修改一个类需要同步修改其他类时。</font>
+ <font style="color:rgb(64, 64, 64);">当需要新增类似功能但不想复制代码时。</font>

---

### <font style="color:rgb(64, 64, 64);">总结</font>
+ **<font style="color:rgb(64, 64, 64);">异曲同工的类</font>**<font style="color:rgb(64, 64, 64);">是代码冗余和维护负担的典型表现。</font>
+ **<font style="color:rgb(64, 64, 64);">重构核心</font>**<font style="color:rgb(64, 64, 64);">：通过继承或组合抽象公共逻辑，隔离差异点。</font>
+ **<font style="color:rgb(64, 64, 64);">适用模式</font>**<font style="color:rgb(64, 64, 64);">：模板方法模式、策略模式或桥接模式。</font>
+ **<font style="color:rgb(64, 64, 64);">关键原则</font>**<font style="color:rgb(64, 64, 64);">：遵循 DRY 原则和 SOLID 原则中的开闭原则、单一职责原则。</font>
