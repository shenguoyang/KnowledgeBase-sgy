# 1、监控性能计数器：
**介绍：**

性能计数器做一个模板导出，写一个脚本把该模板自动导入到电脑的性能监视器中并自动开启监控

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">Processor</font>**

综合/单个核心:   选择综合

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">% Processor Time（% Processor Time）：</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">•描述：表示CPU在采样间隔内用于处理非空闲线程的时间百分比。在多核心系统中，每个核心都有对应的“% Processor Time”计数器，可以监控每个核心的利用率。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">% User Time：</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">•描述：CPU在处理用户模式代码（应用程序）所花费时间的百分比。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">% Privileged Time：</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">•描述：CPU在处理内核模式代码（操作系统和驱动程序）所花费时间的百分比。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">% interupt Time：</font>**

<font style="color:rgb(64, 64, 64);"> 是衡量 CPU 响应和处理中断效率的关键指标</font>

<font style="color:rgb(64, 64, 64);">假设一个网络数据包到达网卡：</font>

1. <font style="color:rgb(64, 64, 64);">网卡触发硬件中断。</font>
2. <font style="color:rgb(64, 64, 64);">CPU 停止当前任务，保存状态，执行网卡驱动中的 ISR 读取数据。</font>
3. <font style="color:rgb(64, 64, 64);">ISR 完成后，CPU 恢复原任务。</font>
4. <font style="color:rgb(64, 64, 64);">整个过程中断时间若为 </font>**<font style="color:rgb(64, 64, 64);">10μs</font>**<font style="color:rgb(64, 64, 64);">，而每秒处理 </font>**<font style="color:rgb(64, 64, 64);">1000 次中断</font>**<font style="color:rgb(64, 64, 64);">，则总中断时间为 </font>**<font style="color:rgb(64, 64, 64);">10ms/s</font>**<font style="color:rgb(64, 64, 64);">，占用了一个核心的 1% 的 CPU 时间。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">Interrupts/sec：</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">•描述：每秒钟CPU处理的硬件中断次数，过高可能表明系统有大量的硬件活动或驱动问题。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">% DPC time：</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">•描述： 它代表了处理器用于处理DPC（延迟过程调用）的时间百分比。</font>

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;"> 如果DPC Time的值持续较高，可能意味着系统存在较多的硬件中断或者DPC处理负担较重，可能导致系统响应变慢或出现延迟。对于多处理器系统，如果单个处理器的DPC Time较高且伴随Processor Time也很高，可能需要进一步调查硬件驱动程序、设备间的交互或者系统配置等因素    </font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">%C1 Time；%C2 Time；%C2 Time</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;"> •C1 State (Clock Gated)： C1状态是最浅的睡眠状态，此时处理器的大部分电路依然保持供电，但是时钟信号可能会被暂停或减慢，以降低能耗。处理器通常在很短的时间内就能从C1状态恢复到全速运行状态。•C2 State (Deep Sleep)： C2状态相对于C1更深的休眠状态，处理器更多的组件被关闭或进入低功耗模式，唤醒时间相对C1状态会长一些，但功耗进一步降低。•C3 State (Deeper Sleep)： C3状态是更深的睡眠级别，处理器会关闭更多不必要的电路，进一步降低功耗。从C3状态唤醒至全速运行通常需要更长的时间，但其节能效果更好。 </font>

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;"> 例如：•如果C3 Time % 较大，意味着处理器长时间处于深度休眠状态，这在轻负载或空闲状态下是正常的，有助于节省电能。   </font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;"> DPC Rate（Deferred Procedure Call Rate）</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">当硬件设备发生中断时，操作系统可能会延迟对中断的完整处理，而安排一个DPC函数在未来某个适当时机执行，以减少中断上下文切换带来的开销，并提高系统的整体响应速度。</font>

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">DPC Rate的高低反映了系统中硬件设备与内核之间异步通信的活跃程度。如果DPC Rate很高，可能意味着系统正面临大量的硬件I/O操作或驱动程序正在进行密集的异步处理。如果DPC Rate持续偏高且伴随着较高的DPC CPU使用时间，可能预示着系统存在I/O瓶颈或驱动程序问题，这可能会影响到系统的整体性能和响应时间。因此，监控和分析DPC Rate对于系统性能优化和故障排查是很重要的。  </font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;"> physical disk  </font>**

1. **<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">% Disk Time：</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">显示磁盘驱动器在采样期间忙于处理I/O请求的时间百分比。高百分比表示磁盘可能过载或存在性能瓶颈。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">2. Average Disk Queue Length：</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">平均排队等待磁盘处理的I/O请求数量。长时间保持高位可能意味着I/O请求堆积，系统可能因为磁盘响应慢而出现延迟</font>**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">。</font>**

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">3. Disk Reads/sec 和 Disk Writes/sec：</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">分别表示每秒钟从磁盘读取和写入操作的次数，用于评估磁盘读写操作的频率和活跃度。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">4. Disk Read Bytes/sec 和 Disk Write Bytes/sec：</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">分别表示每秒钟从磁盘读取和写入的数据量（以字节为单位），用于衡量磁盘读写操作的吞吐量。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">5. Avg. Disk sec/Read 和 Avg. Disk sec/Write：</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">分别表示平均每次读取和写入操作所需的时间（以秒为单位），用来评估磁盘读写操作的响应速度。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">6. Current Disk Queue Length：</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">当前等待磁盘处理的I/O请求数目，可实时了解当前磁盘队列压力。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">7. disk transfers/sec:  </font>**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;"> </font>**<font style="color:rgb(64, 64, 64);">Avg. Disk sec/Transfer:</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">次/秒（I/O operations per second, IOPS）; </font>

+ `**<font style="color:rgb(64, 64, 64);">Disk Transfers/sec</font>**`**<font style="color:rgb(64, 64, 64);"> </font>****<font style="color:rgb(64, 64, 64);">反映磁盘 I/O 频率</font>**<font style="color:rgb(64, 64, 64);">，是评估磁盘压力的核心指标。</font>
+ **<font style="color:rgb(64, 64, 64);">高值不一定表示问题</font>**<font style="color:rgb(64, 64, 64);">，需结合延迟（</font>`<font style="color:rgb(64, 64, 64);">Avg. Disk sec/Transfer</font>`<font style="color:rgb(64, 64, 64);">）和队列长度判断是否过载。</font>
+ <font style="color:rgb(64, 64, 64);">对于数据库、虚拟机等密集 I/O 场景，建议持续监控并设定告警阈值（如 HDD > 150 IOPS 或 SSD > 80% 标称值）。</font>
+ **<font style="color:rgb(64, 64, 64);">每次传输的数据量（Bytes） = </font>**`**<font style="color:rgb(64, 64, 64);">Avg. Disk Bytes/Transfer</font>**`<font style="color:rgb(64, 64, 64);">  
</font><font style="color:rgb(64, 64, 64);">（该计数器直接提供每次 I/O 操作的平均字节数）</font>
+ **<font style="color:rgb(64, 64, 64);">高 IOPS + 低 Bytes/Transfer</font>**<font style="color:rgb(64, 64, 64);"> </font><font style="color:rgb(64, 64, 64);">→ 随机小 I/O 密集型（如数据库 OLTP）。</font>
+ **<font style="color:rgb(64, 64, 64);">低 IOPS + 高 Bytes/Transfer</font>**<font style="color:rgb(64, 64, 64);"> → 顺序大块 I/O（如视频流）。</font>

#### **<font style="color:rgb(64, 64, 64);">hdd示例</font>**
+ <font style="color:rgb(64, 64, 64);">应用程序请求读取 8KB：</font>
    - <font style="color:rgb(64, 64, 64);">若文件是连续存储的，HDD 可能直接读取 2 个连续 4KB 扇区（计为 1 次传输）。</font>
    - <font style="color:rgb(64, 64, 64);">若文件碎片化，可能需要 2 次独立寻道（计为 2 次传输）。</font>

#### **<font style="color:rgb(64, 64, 64);">ssd示例</font>**
+ <font style="color:rgb(64, 64, 64);">应用程序请求写入 4KB：</font>
    - <font style="color:rgb(64, 64, 64);">SSD 可能实际写入 1 个 16KB 闪存页（需先擦除整个块，再写入新数据）。</font>
    - <font style="color:rgb(64, 64, 64);">但操作系统仍计为 1 次传输（逻辑 I/O 完成）。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">8. </font>**`**<font style="color:rgb(64, 64, 64);">Split IO/Sec</font>**`

### **<font style="color:rgb(64, 64, 64);">为什么需要拆分 I/O？</font>**
#### **<font style="color:rgb(64, 64, 64);">(1) 物理存储的固有限制</font>**
+ **<font style="color:rgb(64, 64, 64);">HDD</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - <font style="color:rgb(64, 64, 64);">单次 I/O 通常不能跨磁道（Track）或柱面（Cylinder）。</font>
    - <font style="color:rgb(64, 64, 64);">若请求的数据块跨越多个磁道，需拆分为多次寻道操作。</font>
+ **<font style="color:rgb(64, 64, 64);">SSD</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - <font style="color:rgb(64, 64, 64);">闪存页（Page，通常 4KB~16KB）是读写最小单位，块（Block，通常 256KB~4MB）是擦除最小单位。</font>
    - <font style="color:rgb(64, 64, 64);">若 I/O 请求未对齐闪存页或需修改部分数据，SSD 控制器可能拆分请求。</font>

#### **<font style="color:rgb(64, 64, 64);">(2) 文件系统与磁盘布局</font>**
+ **<font style="color:rgb(64, 64, 64);">簇/扇区不对齐</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - <font style="color:rgb(64, 64, 64);">若文件系统簇大小为 4KB，但 I/O 请求 8KB 且偏移量为 2KB，则会跨越 3 个簇（2KB + 4KB + 2KB），触发拆分。</font>
+ **<font style="color:rgb(64, 64, 64);">分区边界跨越</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - <font style="color:rgb(64, 64, 64);">I/O 请求跨越磁盘分区边界时（罕见）。</font>

#### **<font style="color:rgb(64, 64, 64);">(3) 驱动程序与缓存策略</font>**
+ **<font style="color:rgb(64, 64, 64);">RAID 条带化（Striping）</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - <font style="color:rgb(64, 64, 64);">若 I/O 请求跨越多块磁盘的条带（Stripe），RAID 控制器会拆分请求并并行分发。</font>
+ **<font style="color:rgb(64, 64, 64);">写入缓存合并失败</font>**<font style="color:rgb(64, 64, 64);">：</font>
    - <font style="color:rgb(64, 64, 64);">非连续的小 I/O 无法合并，可能被拆分为独立操作。</font>



**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;"> Memory</font>**

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;"> 1. Available MBytes：</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">•描述：系统当前可供进程立即使用的物理内存总量（不包括系统保留区域和虚拟内存）。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">2. Committed Bytes：</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">•描述：系统为进程分配并承诺使用的虚拟内存总量，包括物理内存和分页文件（页面文件）。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">3. Page Faults/sec：</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">•描述：每秒钟系统发生的页面错误次数，包括硬错误（需要从磁盘读取数据）和软错误（内存中的页面只是被移动，而不是从磁盘读取）。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">4. Pages/sec：page input/sec: page output/sec:</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">•描述：每秒钟系统进行的页面换入（从磁盘读取到内存）和换出（从内存写入到磁盘）操作的总数。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">5. Pool Nonpaged Bytes 和 Pool Paged Bytes：</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">•描述：分别表示系统非分页池和分页池的当前使用量，非分页池中的内存不能被换出到磁盘，而分页池中的内存可以。这两块属于</font>**<font style="color:rgb(64, 64, 64);">内核内存池</font>**<font style="color:rgb(64, 64, 64);">是操作系统为 </font>**<font style="color:rgb(64, 64, 64);">内核模式组件</font>**<font style="color:rgb(64, 64, 64);">（如驱动程序、文件系统、网络协议栈）预分配的内存区域，</font>

#### <font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">    Pool Nonpaged Bytes 过高</font>
+ <font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">可能原因</font><font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">：</font>
    - <font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">驱动程序内存泄漏（如显卡驱动、杀毒软件）。</font>
    - <font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">恶意软件占用内核内存。</font>
+ 

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">6. Cache Bytes：</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">•描述：系统缓存所使用的内存大小，包括文件系统缓存、映射文件和页面缓冲池。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">7. Transition Faults/sec：</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">•描述：每秒钟系统发生的页面错误中，那些原本在分页文件中，但由于近期被访问过而重新加载到物理内存中的次数。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">8. % Committed Bytes In Use：</font>**

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">•描述：已提交内存（Committed Bytes）与系统物理内存总量的百分比，用于评估物理内存的压力。通过监控这些计数器，可以了解系统内存的使用效率、内存管理活动、是否存在内存瓶颈等问题，从而帮助进行性能优化或规划系统升级。  </font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">Process</font>**

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">Processor Time</font>**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">:</font>

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">表示指定进程占用CPU总时间的百分比，即进程在其生命周期内所使用的CPU时间除以总的CPU时间。对于多核系统，该值可能超过100%，因为它是所有CPU核心上该进程所消耗时间的总和。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">Working Set</font>**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">:</font>

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">表示进程当前在物理内存中总共占用的总量，包括独享和共享的物理内存，不包括分页文件中交换出的内存。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">Private Bytes</font>**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">:</font>

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">表示进程专有的虚拟地址空间中由该进程分配并且目前尚未释放的字节数。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">Virtual Bytes</font>**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">:</font>

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">表示进程已分配的虚拟地址空间总量，包括私有字节和共享字节。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">Thread Count</font>**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">:</font>

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">指的是进程中活动线程的数量。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">Page Faults/sec</font>**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">:</font>

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">表示每秒钟由于进程导致的页面错误次数，包括硬错误（必须从磁盘读取数据）和软错误（在内存中重新定位页面）。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">IO Read Operations/sec</font>**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;"> 和 </font>**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">IO Write Operations/sec</font>**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">:</font>

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">分别表示进程每秒执行的磁盘读操作和写操作次数。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">IO Data Operations/sec</font>**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">:</font>

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">表示每秒进程执行的磁盘读写操作的数据量。</font>

**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">Handle Count</font>**<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">:</font>

<font style="color:rgb(23, 26, 29);background-color:#FFFFFF;">指的是进程当前打开的句柄数量，包括文件、同步对象、套接字等资源。</font>

### **1. Processor 对象（CPU 分析）**
| 计数器名称 | 含义 | 报警阈值 | 与其他计数器的数学关系 |
| --- | --- | --- | --- |
| **% Processor Time** | CPU 执行非空闲线程的时间百分比 | >80%（持续） | `= % User Time + % Privileged Time` |
| **% User Time** | 用户模式下运行应用程序代码的时间百分比 | >70%（特定进程） | 子集于 % Processor Time |
| **% Privileged Time** | 内核模式下运行操作系统代码的时间百分比 | >30%（持续） | `= % Interrupt Time + % DPC Time + % Hypervisor Time + 其他内核时间` |
| **% Interrupt Time** | 处理硬件中断的时间百分比 | >5%（服务器）   >2%（桌面） | 子集于 % Privileged Time |
| **% DPC Time** | 处理延迟过程调用的时间百分比 | >3%（持续） | 子集于 % Privileged Time |
| **% Hypervisor Time** | Hyper-V 虚拟化开销时间（启用虚拟化时） | >10%（虚拟机宿主机） | 额外开销，不计入 % Privileged Time |
| **% Idle Time** | CPU 空闲时间百分比 | <20%（高负载系统） | `= 100% - % Processor Time` |
| **% C1/C2/C3 Time** | C1/C2 低功耗状态时间（节能模式） | C3 >50%（性能劣化） | `% Processor Time + ΣC-State% = 100%` |
| **% RC3/RC6 Time** | 现代 CPU 深度节能状态时间（Haswell+） | RC6 >60%（性能劣化） | 包含在 % Idle Time 中 |
| **Processor Frequency** | 当前 CPU 频率（GHz） | < 基础频率（节能异常） | 与 % Processor Time 正相关 |
| **Processor Power** | CPU 功耗（W） | > TDP 的 90%（散热风险） | 与 Processor Frequency³ 正相关 |
| **Processor Performance** | 性能状态百分比（P-State） | <70%（频率受限） | `= 当前频率 / 最大睿频` |
| **APCs Received/sec** | 异步过程调用接收率 | >1000（高并发系统） | 与 Context Switches/sec 正相关 |
| **DPCs Queued/sec** | 延迟过程调用排队速率 | >200（网络/存储设备） | `% DPC Time ≈ DPCs Queued/sec * Avg. DPC 处理时间` |


---

### **2. Process 对象（进程分析）**
| 计数器名称 | 含义 | 报警阈值 | 与其他计数器的数学关系 |
| --- | --- | --- | --- |
| **% Processor Time** | 进程占用 CPU 百分比 | >70%（持续） | `Σ(所有进程) ≈ Processor(_Total)\% Processor Time` |
| **% Privileged Time** | 进程在内核模式的时间百分比 | >20%（应用进程） | 子集于 % Processor Time |
| **Working Set** | 进程占用的物理内存量（字节） | > 总内存的 30% | `Σ(所有进程) ≤ Memory\Available Bytes` |
| **Working Set - Private** | 进程私有内存量（不可共享） | 持续增长（泄漏） | ≤ Working Set |
| **Virtual Bytes** | 进程使用的虚拟地址空间总量 | > 物理内存的 200% | `= Private Bytes + Mapped Files + Shared Memory` |
| **Private Bytes** | 进程申请的私有内存量（含分页文件） | 持续增长（泄漏） | `≈ Virtual Bytes - (共享内存+映射文件)` |
| **Page Faults/sec** | 进程缺页中断率 | >1000（持续） | `硬缺页率 = Page File Bytes / Virtual Bytes * Page Faults/sec` |
| **Handle Count** | 进程打开的句柄数 | >10,000（普通进程） | `Σ(所有进程) = System\Handles` |
| **Thread Count** | 进程创建的线程数 | >500（普通进程） | `Σ(所有进程) = System\Threads` |
| **IO Data Bytes/sec** | 进程 I/O 吞吐量（读写总和） | >500MB/s（磁盘瓶颈） | `= IO Read Bytes/sec + IO Write Bytes/sec` |
| **IO Other Operations/sec** | 非读写操作率（控制命令） | >1000（存储进程） | 与 PhysicalDisk\Current Disk Queue Length 相关 |
| **Page File Bytes** | 进程在分页文件中的占用量 | > Working Set（内存不足） | 高值导致 Memory\Pages/sec 上升 |
| **Elapsed Time** | 进程运行总时间（秒） | N/A（基线对比） | 结合 % Processor Time 计算 CPU 占用率 |


---

### **3. Memory 对象（内存分析）**
| 计数器名称 | 含义 | 报警阈值 | 与其他计数器的数学关系 |
| --- | --- | --- | --- |
| **Available MBytes** | 可用物理内存量（MB） | < 总内存的 10% 或 <100MB | `= Total Physical RAM - (Standby Cache + Modified Page List + Process Working Sets)` |
| **Committed Bytes** | 已提交虚拟内存量（物理+分页文件） | > Commit Limit 的 90% | `= Memory\Pool Paged Bytes + Process(_Total)\Private Bytes` |
| **Commit Limit** | 最大可提交内存量（物理+分页文件） | N/A（参考值） | `= Physical RAM + Page File Size - Reserved` |
| **Pages/sec** | 页交换速率（硬缺页） | >100（服务器）   >50（桌面） | `= Page Reads/sec + Page Writes/sec` |
| **Page Reads/sec** | 从磁盘读取页的速率 | >50（性能劣化） | 高值导致 PhysicalDisk\Avg. Disk sec/Read 上升 |
| **Pool Nonpaged Bytes** | 内核非分页池大小 | 持续增长（泄漏） | 异常增长时 Available MBytes 骤降 |
| **Pool Paged Bytes** | 内核分页池大小 | 持续增长 | 高值 + Pages/sec 高 → 内存压力大 |
| **Cache Bytes** | 系统文件缓存总量 | N/A（动态调整） | `= Standby Cache + Modified Page List` |
| **Cache Faults/sec** | 文件缓存未命中率 | > Cache Bytes/10,000 | 高值导致 FileSystem\Metadata Reads/sec 上升 |
| **Free System Page Table Entries** | 空闲页表项数量 | <5,000（x64 风险） | 与 Virtual Bytes 负相关 |
| **Page Writes/sec** | 页写入磁盘速率 | >100（内存压力） | `= Modified Page List Flushes + Mapped File Writes` |
| **Transition Faults/sec** | 软缺页中断率（从Standby Cache恢复） | >10,000（缓存效率低） | `总缺页 = Page Faults/sec + Transition Faults/sec` |

### **4. System 对象（系统全局）**
| 计数器名称 | 含义 | 报警阈值 | 与其他计数器的数学关系 |
| --- | --- | --- | --- |
| **Processor Queue Length** | CPU 等待队列长度（所有核心） | > 2 * CPU 逻辑核心数 | 当 Processor(_Total)% Processor Time >90% 时此值高 → CPU 瓶颈 |
| **Context Switches/sec** | 线程切换频率 | >20,000/核心（x86）   >50,000/核心（x64） | `= Thread dispatches/sec * Avg. Context Switches per Dispatch` |
| **System Calls/sec** | 应用程序调用内核服务的频率 | >100,000/秒 | `≈ Processor(_Total)\% Privileged Time * CPU 频率 / 10` |
| **File Control Operations/sec** | 文件控制操作率（打开/关闭等） | >10,000（文件服务器） | 与 Process(explorer)\Handle Count 相关 |
| **Registry Queries/sec** | 注册表查询速率 | >5,000（配置密集型应用） | 高值导致 Process(svchost)% Processor Time 上升 |
| **Alignment Fixups/sec** | 内存对齐修复次数（x86兼容模式） | >0（应用兼容问题） | 非零值表示存在 32 位应用兼容性问题 |
| **Exception Dispatches/sec** | 异常分发速率 | >100（应用错误） | 高值导致 Process(w3wp)% Processor Time 上升 |
| **Floating Emulations/sec** | 浮点运算模拟次数（无FPU硬件） | >0（性能警告） | 每次模拟消耗 ≈ 100 条普通指令 |
| **Threads** | 系统活动线程总数 | >15,000（普通服务器） | `= Σ(Process(*)\Thread Count)` |
| **Processes** | 活动进程总数 | 突然翻倍（异常） | 与 Process 对象实例数一致 |
| **Handles** | 系统句柄总数 | >50,000（资源泄漏） | `= Σ(Process(*)\Handle Count)` |
| **Objects** | 内核对象总数 | >100,000（资源压力） | 包含 Events, Semaphores, Mutexes 等 |

### **5. PhysicalDisk 对象（磁盘分析）**
| 计数器名称 | 含义 | 报警阈值 | 与其他计数器的数学关系 |
| --- | --- | --- | --- |
| **% Disk Time** | 磁盘忙于读写的时间百分比 | >90%（持续） | `= % Disk Read Time + % Disk Write Time` |
| **% Disk Read Time** | 磁盘读操作时间百分比 | >70%（读密集型） | 子集于 % Disk Time |
| **% Disk Write Time** | 磁盘写操作时间百分比 | >70%（写密集型） | 子集于 % Disk Time |
| **% Idle Time** | 磁盘空闲时间百分比 | <10%（高负载） | `= 100% - % Disk Time` |
| **Avg. Disk Queue Length** | 等待磁盘的请求平均数 | >2（单个磁盘）   > 2 * 磁盘数（RAID） | `= Disk Transfers/sec * Avg. Disk sec/Transfer` |
| **Avg. Disk Read Queue Length** | 读请求队列长度 | >1（性能劣化） | 子集于 Avg. Disk Queue Length |
| **Avg. Disk Write Queue Length** | 写请求队列长度 | >1（性能劣化） | 子集于 Avg. Disk Queue Length |
| **Disk Bytes/sec** | 磁盘吞吐量（字节/秒） | >85% 磁盘理论带宽 | `= Disk Read Bytes/sec + Disk Write Bytes/sec` |
| **Disk Read Bytes/sec** | 读吞吐量 | > SSD 500MB/s   > HDD 100MB/s | 高值 + Memory\Pages/sec 高 → 内存不足 |
| **Avg. Disk sec/Read** | 读操作平均延迟 | >15ms（HDD）   >5ms（SSD） | `% Disk Read Time ≈ Disk Reads/sec * Avg.sec/Read * 100%` |
| **Avg. Disk sec/Write** | 写操作平均延迟 | >20ms（HDD）   >10ms（SSD） | `% Disk Write Time ≈ Disk Writes/sec * Avg.sec/Write * 100%` |
| **Split IO/sec** | 拆分 I/O 请求次数（文件碎片） | >10% of Disk Transfers/sec | 高值需检查 FileSystem\File Fragments |
| **Disk Writes/sec** | 写操作频率 | > SSD 10,000   > HDD 300 | 与 Memory\Page Writes/sec 正相关 |
| **Current Disk Queue Length** | 当前未完成请求数（瞬时值） | > Avg. Disk Queue Length * 2 | 突发性 I/O 压力的直接指标 |
