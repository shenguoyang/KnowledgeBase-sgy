**1、类关系：**

依赖<关联<聚合<组合<继承<实现


**1.1****依赖关系：**

是一种使用的关系，即一个类的实现需要另一个类的协助，即use关系

依赖三种形式：局部变量；函数参数；静态方法。

class Car

{

public:

	static void doSomething1() {};

	void doSomething2() {};

};

class Person

{

public:

	//形式一：函数参数

	void drive1(Car car) 

	{

		car.doSomething2();

	};

	//形式二：局部变量

	void drive2() 

	{ 

		Car* car = new Car(); 

		car->doSomething2();

	};

	//形式三：静态方法

	void drive3() 

	{

		Car::doSomething1();

	};

private:

};

**1.2****关联关系：**

两个类在依赖的基础上增加了一层关系，即一个类作为另一个类的属性类型，即has a关系。

从关联方向可分：分为单向关联和双向关联

从关联的多重性可分：一对一；一对多；

class Student;

class Teacher

{

public:

	void setStudent(Student student) { m_studentList.push_back(student); };

private:

	std::vector<Student> m_studentList;

};

class Student

{

public:

	void setCourse(Grade grade) { m_grade = grade; };

	void setTeacher(Teacher teacher) { m_teacher = teacher; };

private:

	Grade m_grade;  //班级

	Teacher m_teacher;//班主任老师

其中班主任老师和学生是双向关联关系，学生和课程是单向关联关系。

其中学生与老师是一对多的关系，学生与班级是一对一关系。

**依赖和关联区别：**

一个类临时使用到另一个类，类关系至少是依赖关系

一个类作为另一个类的成员对象时，类关系至少是关联关系。


**1.3****聚合关系：**

两个类在关联的基础上又增加了一层关系，即整体和部分。即has a

**聚合和关联的区别**

关联关系所涉及的两个类是同一个层次上的，比如学生和课程，学生不是由课程组成的

聚合关系是两个类是不平等层次上，一个代表整体，一个代表部分

**1.4****组合关系：**

两个类在聚合的基础上又增加了一层关系，即相同生命周期。即contain a

class House

{

public:

	House() :m_brick(new Brick) {};

	~House() { delete m_brick; }

	void setVideo(Television television) { m_television = television; };

private:

	Television m_television;  //电视机

	Brick* m_brick;           //砖头

};

其中电视机和房子就是聚合关系，砖头与房子是组合关系。

**聚合和组合的区别**

都是整体和部分的关系，组合关系两个类同生命周期，聚合可以不同生命周期

**1.5****继承关系：**

**一个类继承另一个类，则两个类是继承关系，即****is a**

**1.6****实现关系：**

**一个类继承只由纯虚函数构成的类，则两个类是实现关系**
