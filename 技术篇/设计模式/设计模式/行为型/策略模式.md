### 1. 传统支付实现痛点分析
#### 1.1 业务场景描述
某电商平台需要支持多种支付方式：

+ 支付宝支付（即时到账）
+ 微信支付（二维码生成）
+ 信用卡支付（3D安全验证）  
未来需扩展PayPal、数字货币等支付方式

#### 1.2 问题代码实现
```cpp
class PaymentService {
public:
enum Method { Alipay, WeChat, CreditCard };

void ProcessPayment(Method method, float amount) {
    // 公共预处理
    ValidateAmount(amount);
    LogPaymentStart();

    // 差异化支付逻辑
    switch(method) {
        case Alipay:
            CallAlipayAPI(amount);
            break;
        case WeChat:
            GenerateQRCode(amount);
            break;
        case CreditCard:
            Verify3DSecure(amount);
            break;
    }

    // 公共后处理
    LogPaymentEnd();
}

private:
void ValidateAmount(float a) { /* 金额校验 */ }
void LogPaymentStart() { /* 日志记录 */ }
void LogPaymentEnd() { /* 日志记录 */ }

// 各支付方式实现
void CallAlipayAPI(float a) { /* 支付宝接口调用 */ }
void GenerateQRCode(float a) { /* 生成微信二维码 */ }
void Verify3DSecure(float a) { /* 信用卡3D验证 */ }
};

// 客户端调用
int main() {
    PaymentService service;

    // 支付宝支付
    service.ProcessPayment(PaymentService::Alipay, 100.0f);

    // 微信支付
    service.ProcessPayment(PaymentService::WeChat, 200.0f);

    // 新增支付方式需要修改枚举和switch-case
    return 0;
}
```

#### 1.3 五大缺陷与后果
| **缺陷类型** | **代码表现** | **实际后果** |
| --- | --- | --- |
| **扩展性** | 新增支付方式需修改枚举和switch-case | 支付核心模块每周需要重新发布 |
| **耦合性** | 支付逻辑与业务处理深度耦合 | 修改微信支付影响支付宝功能测试 |
| **复用性** | 金额校验和日志记录重复出现 | 修改日志格式需改动所有支付分支 |
| **维护性** | 500行巨型类随业务增长持续膨胀 | 定位支付BUG平均耗时2小时+ |
| **性能** | 每次支付都需执行switch判断 | 高峰时段支付延迟增加300ms |

### 2. 策略模式解决方案
#### 2.1 重构后的代码
```cpp
// 策略接口
class IPaymentStrategy {
public:
virtual ~IPaymentStrategy() = default;
virtual void Execute(float amount) = 0;
};

// 具体策略实现
class AlipayStrategy : public IPaymentStrategy {
public:
void Execute(float amount) override {
    cout << "调用支付宝API，金额：" << amount << endl;
    // 实际接口调用逻辑
}
};

class WeChatStrategy : public IPaymentStrategy {
public:
void Execute(float amount) override {
    cout << "生成微信支付二维码，金额：" << amount << endl;
    // 二维码生成逻辑
}
};

// 上下文类
class PaymentProcessor {
unique_ptr<IPaymentStrategy> strategy;

void CommonPreProcess(float amount) {
    if(amount <= 0) throw invalid_argument("金额必须大于0");
    cout << "[日志] 支付开始，时间戳：" << time(nullptr) << endl;
}

void CommonPostProcess() {
    cout << "[日志] 支付结束" << endl;
}

public:
explicit PaymentProcessor(unique_ptr<IPaymentStrategy> s) 
: strategy(move(s)) {}

void Process(float amount) {
    CommonPreProcess(amount);
    strategy->Execute(amount);
    CommonPostProcess();
}

void SwitchStrategy(unique_ptr<IPaymentStrategy> newStrategy) {
    strategy = move(newStrategy);
}
};

// 客户端调用
int main() {
    // 支付宝支付
    auto alipay = make_unique<AlipayStrategy>();
    PaymentProcessor processor1(move(alipay));
    processor1.Process(100.0f);

    // 微信支付
    auto wechat = make_unique<WeChatStrategy>();
    PaymentProcessor processor2(move(wechat));
    processor2.Process(200.0f);

    // 动态切换支付方式
    PaymentProcessor dynamicProcessor(make_unique<AlipayStrategy>());
    dynamicProcessor.Process(300.0f);
    dynamicProcessor.SwitchStrategy(make_unique<WeChatStrategy>());
    dynamicProcessor.Process(150.0f);

    return 0;
}
```

# **6.****与其他相近模式区别**


从工厂模式的代码中可以看到 工厂模式主要是返回的接口实现类的实例化对象，最后返回的结果是接口实现类中的方法，而策略模式是在实例化策略模式的时候已经创建好了，我们可以再策略模式中随意的拼接重写方法，而工厂模式是不管方法的拼接这些的，他只关注最后的结果，不注重过程，而策略模式注重的是过程。如下：


策略模式的run方法可以组合，简单工厂模式不可以，它返回的是对象

****


在策略模式中，结合**函数组合**可以赋予上下文类更灵活的流程控制能力。我们可以通过允许用户自定义预处理、核心策略（如排序）和后处理逻辑，实现高度可定制的执行流程。以下是具体实现：

---

### 场景需求
假设我们需要一个支持以下扩展功能的排序器：

1. **预处理**：如数据校验、去重
2. **核心排序**：不同的排序算法
3. **后处理**：如结果验证、日志记录

---

### C++实现（支持函数组合）
```cpp
#include <iostream>
#include <vector>
#include <functional>
#include <memory>

// 策略接口
class SortingStrategy {
public:
    virtual void sort(std::vector<int>& data) const = 0;
    virtual ~SortingStrategy() = default;
};

// 具体策略类
class QuickSort : public SortingStrategy {
public:
    void sort(std::vector<int>& data) const override {
        std::cout << "[核心策略] 执行快速排序\n";
        // 实际排序逻辑...
    }
};

// 上下文类（支持函数组合）
class AdvancedSorter {
private:
    std::unique_ptr<SortingStrategy> strategy;
    std::function<void(std::vector<int>&)> preprocessor = nullptr;
    std::function<void(const std::vector<int>&)> postprocessor = nullptr;

public:
    explicit AdvancedSorter(std::unique_ptr<SortingStrategy> strat)
        : strategy(std::move(strat)) {}

    // 设置预处理函数
    void setPreprocessor(std::function<void(std::vector<int>&)> func) {
        preprocessor = func;
    }

    // 设置后处理函数
    void setPostprocessor(std::function<void(const std::vector<int>&)> func) {
        postprocessor = func;
    }

    // 执行组合流程
    void executeSort(std::vector<int>& data) {
        if(preprocessor) {
            std::cout << "[阶段1] 执行预处理\n";
            preprocessor(data);  // 执行预处理
        }

        std::cout << "[阶段2] 执行核心排序\n";
        strategy->sort(data);     // 执行核心策略

        if(postprocessor) {
            std::cout << "[阶段3] 执行后处理\n";
            postprocessor(data); // 执行后处理
        }
    }
};

// 示例预处理函数：数据去重
void deduplicate(std::vector<int>& data) {
    std::cout << "--> 预处理：移除重复元素\n";
    // 实际去重逻辑...
}

// 示例后处理函数：结果验证
void validateSort(const std::vector<int>& data) {
    std::cout << "--> 后处理：验证排序结果\n";
    // 实际验证逻辑...
}

int main() {
    std::vector<int> data = {5, 2, 2, 7, 1, 3};

    // 创建带函数组合的排序器
    AdvancedSorter sorter(std::make_unique<QuickSort>());
    
    // 自定义处理流程
    sorter.setPreprocessor(deduplicate);
    sorter.setPostprocessor([](const std::vector<int>& d) {  // 使用lambda表达式
        std::cout << "--> 后处理：写入日志（元素数量：" << d.size() << "）\n";
    });
    sorter.setPostprocessor(validateSort);  // 可以叠加多个后处理

    // 执行完整流程
    sorter.executeSort(data);

    return 0;
}
```

### 关键设计解析
1. **灵活的函数接口**：
    - 使用`std::function`定义预处理(`void(std::vector<int>&)`)和后处理(`void(const std::vector<int>&)`)接口
    - 兼容自由函数、lambda表达式、函数对象等多种形式
2. **可叠加的处理流程**：
    - 预处理和后处理均可设置多个（示例中后处理叠加了日志和验证）
    - 实际开发中可用`std::vector<std::function<...>>`存储多个处理函数
3. **执行流程控制**：

```cpp
void executeSort(std::vector<int>& data) {
    // 1. 执行所有预处理（按添加顺序）
    for(auto& pre : preprocessors) pre(data);
    
    // 2. 执行核心策略
    strategy->sort(data);
    
    // 3. 执行所有后处理（按添加顺序）
    for(auto& post : postprocessors) post(data);
}
```

将策略模式与简单工厂结合，确实能**进一步解耦对象的创建过程**，使客户端无需直接依赖具体策略类，提升代码的可维护性和扩展性。以下是优化后的C++实现：

### 改进思路
1. **工厂封装创建逻辑**：通过工厂类集中管理策略对象的实例化。
2. **客户端与策略解耦**：客户端只需指定策略类型，无需知晓具体类名。
3. **统一入口管理**：新增策略时，仅需扩展工厂类，无需修改客户端代码。

### C++代码示例（结合简单工厂）
```cpp
#include <iostream>
#include <memory>
#include <stdexcept>

// 策略接口
class SortingStrategy {
public:
    virtual void sort(int* data, int size) const = 0;
    virtual ~SortingStrategy() = default;
};

// 具体策略：快速排序
class QuickSort : public SortingStrategy {
public:
    void sort(int* data, int size) const override {
        std::cout << "使用快速排序算法\n";
        // 具体实现...
    }
};

// 具体策略：冒泡排序
class BubbleSort : public SortingStrategy {
public:
    void sort(int* data, int size) const override {
        std::cout << "使用冒泡排序算法\n";
        // 具体实现...
    }
};

// 策略类型枚举（明确可选的策略）
enum class StrategyType {
    QuickSort,
    BubbleSort
};

// 简单工厂类
class StrategyFactory {
public:
    static std::unique_ptr<SortingStrategy> createStrategy(StrategyType type) {
        switch (type) {
            case StrategyType::QuickSort:
                return std::make_unique<QuickSort>();
            case StrategyType::BubbleSort:
                return std::make_unique<BubbleSort>();
            default:
                throw std::invalid_argument("无效的策略类型");
        }
    }
};

// 上下文类（排序器）
class Sorter {
private:
    std::unique_ptr<SortingStrategy> strategy;

public:
    explicit Sorter(StrategyType type) {
        setStrategy(type);
    }

    void setStrategy(StrategyType type) {
        strategy = StrategyFactory::createStrategy(type);
    }

    void executeSort(int* data, int size) const {
        strategy->sort(data, size);
    }
};

int main() {
    int arr[] = {5, 2, 7, 1, 3};
    constexpr int size = sizeof(arr)/sizeof(arr[0]);

    // 通过工厂创建排序器，指定初始策略类型
    Sorter sorter(StrategyType::QuickSort);
    sorter.executeSort(arr, size);  // 输出：使用快速排序算法

    // 动态切换策略类型
    sorter.setStrategy(StrategyType::BubbleSort);
    sorter.executeSort(arr, size);  // 输出：使用冒泡排序算法

    return 0;
}
```

---

### 改进后的优势
1. **创建逻辑集中化**：
    - 所有策略实例化由`StrategyFactory`统一管理。
    - 新增策略时，只需**修改工厂类**，无需调整客户端代码。
2. **类型安全**：
    - 使用`enum class`明确策略类型，避免字符串硬编码或魔法值的风险。
3. **错误处理**：
    - 工厂中可对无效类型抛出异常，增强鲁棒性。
4. **客户端零依赖**：
    - 客户端仅依赖`StrategyType`枚举，与具体策略类完全解耦。

### 总结
+ **适用场景**：策略类型相对固定且需要集中管理创建逻辑时，简单工厂是理想选择。
+ **更高灵活性**：若需动态注册策略（如插件机制），可进一步结合**注册表模式**或**抽象工厂**。
+ **平衡原则**：在“开闭原则”与“复杂度”间权衡，简单工厂在多数场景下已足够。

### 1. **策略 + 装饰器模式：增强策略功能**
**目标**：在不修改原有策略类的情况下，动态添加额外功能（如日志、性能统计）。

```cpp
#include <iostream>
#include <memory>

// 基础策略接口
class CompressionStrategy {
public:
    virtual void compress(const std::string& file) = 0;
    virtual ~CompressionStrategy() = default;
};

// 具体策略：ZIP压缩
class ZipCompression : public CompressionStrategy {
public:
    void compress(const std::string& file) override {
        std::cout << "使用ZIP压缩文件: " << file << "\n";
    }
};

// 装饰器基类：增强策略功能
class CompressionDecorator : public CompressionStrategy {
protected:
    std::unique_ptr<CompressionStrategy> wrapped;
public:
    explicit CompressionDecorator(std::unique_ptr<CompressionStrategy> strat)
        : wrapped(std::move(strat)) {}
};

// 具体装饰器：添加日志记录
class LoggingDecorator : public CompressionDecorator {
public:
    using CompressionDecorator::CompressionDecorator;
    void compress(const std::string& file) override {
        std::cout << "[日志] 开始压缩文件: " << file << "\n";
        wrapped->compress(file);
        std::cout << "[日志] 压缩完成\n";
    }
};

int main() {
    auto strategy = std::make_unique<LoggingDecorator>(
        std::make_unique<ZipCompression>()
    );
    strategy->compress("data.txt");
    /* 输出：
       [日志] 开始压缩文件: data.txt
       使用ZIP压缩文件: data.txt
       [日志] 压缩完成
    */
}
```

### 2. **策略 + 组合模式：复合策略**
**目标**：将多个策略组合为一个复合策略，按顺序执行。

```cpp
class DataProcessor {
public:
    virtual void process(std::vector<int>& data) = 0;
    virtual ~DataProcessor() = default;
};

// 具体策略：过滤负数
class NegativeFilter : public DataProcessor {
public:
    void process(std::vector<int>& data) override {
        data.erase(std::remove_if(data.begin(), data.end(), 
            [](int x) { return x < 0; }), data.end());
    }
};

// 具体策略：数据标准化
class Normalizer : public DataProcessor {
public:
    void process(std::vector<int>& data) override {
        for(auto& x : data) x *= 10;
    }
};

// 复合策略：按顺序执行子策略
class CompositeProcessor : public DataProcessor {
    std::vector<std::unique_ptr<DataProcessor>> processors;
public:
    void addProcessor(std::unique_ptr<DataProcessor> proc) {
        processors.push_back(std::move(proc));
    }
    void process(std::vector<int>& data) override {
        for(auto& p : processors) p->process(data);
    }
};

// 使用示例
CompositeProcessor pipeline;
pipeline.addProcessor(std::make_unique<NegativeFilter>());
pipeline.addProcessor(std::make_unique<Normalizer>());
std::vector<int> data = {1, -2, 3};
pipeline.process(data); // 结果: {10, 30}
```

### 4. **运行时策略自动选择**
**目标**：根据输入特征动态选择最优策略。

```cpp
class SortStrategy {
public:
    virtual void sort(std::vector<int>&) = 0;
};

class QuickSort : public SortStrategy { /*...*/ };
class BubbleSort : public SortStrategy { /*...*/ };

class AutoSorter {
    std::unique_ptr<SortStrategy> selectStrategy(const std::vector<int>& data) {
        if(data.size() > 1000) return std::make_unique<QuickSort>();
        else return std::make_unique<BubbleSort>();
    }
public:
    void sort(std::vector<int>& data) {
        auto strategy = selectStrategy(data);
        strategy->sort(data);
    }
};
```

### 5. **策略 + 命令模式**
**目标**：将策略封装为可撤销的操作。

```cpp
class Command {
public:
    virtual void execute() = 0;
    virtual void undo() = 0;
};

class SortCommand : public Command {
    std::vector<int> backup;
    std::unique_ptr<SortStrategy> strategy;
    std::vector<int>& target;
public:
    SortCommand(std::vector<int>& data, std::unique_ptr<SortStrategy> strat)
        : target(data), strategy(std::move(strat)) {}
    void execute() override {
        backup = target;
        strategy->sort(target);
    }
    void undo() override {
        target = backup;
    }
};
```

### 6. **基于函数对象的策略（C++特化）**
**目标**：利用`std::function`和模板实现轻量级策略。

```cpp
template<typename T>
class MathOperation {
    std::function<T(T, T)> strategy;
public:
    explicit MathOperation(std::function<T(T, T)> op) : strategy(op) {}
    T compute(T a, T b) { return strategy(a, b); }
};

// 使用示例
MathOperation<int> add([](int a, int b) { return a + b; });
MathOperation<double> multiply([](double a, double b) { return a * b; });
std::cout << add.compute(3, 4);      // 输出7
std::cout << multiply.compute(2.5, 4); // 输出10.0
```

### 变种选择指南
| **变种类型** | **适用场景** | **优势** |
| --- | --- | --- |
| 策略+装饰器 | 需要动态添加日志、监控等横切关注点 | 功能扩展不影响核心逻辑 |
| 策略+组合 | 多步骤流水线处理 | 灵活组装复杂流程 |
| 策略+模板方法 | 算法框架固定但部分步骤可变 | 保证结构统一性 |
| 自动策略选择 | 根据运行时数据智能决策 | 提升系统自适应能力 |
| 策略+命令 | 需要支持撤销/重做操作 | 增强操作可逆性 |
| 函数对象策略 | 简单策略且需要极致性能 | 避免虚函数开销 |


根据具体需求选择合适的变种，可在保持策略模式核心优点的同时，解决特定场景下的扩展性问题。
