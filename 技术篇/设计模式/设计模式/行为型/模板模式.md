**不完全一样，但关系非常密切！**

## 🔍 核心区别
### 模板模式（Template Method）
+ **定义**：在基类中定义算法的骨架，允许子类重写特定步骤
+ **实现方式**：通常使用**虚函数**（virtual）
+ **调用方式**：子类重写后，通过基类指针/引用调用

### NVI（Non-Virtual Interface，非虚拟接口）模式
+ **定义**：一种更严格的模板模式实现规范
+ **核心思想**：**基类的公有方法永远不应该是虚函数**
+ **实现方式**：
    - 公有方法都是**非虚的**（模板方法）
    - 虚函数都应该是**私有的**或**保护的**
    - 通过公有非虚方法调用私有虚方法

---

## 📊 对比示例
### 1. 普通模板模式（虚函数在public）
```cpp
class CaffeineBeverage {
public:
    virtual void brew() = 0;  // 公有虚函数
    virtual void addCondiments() = 0;
    
    void prepareRecipe() {     // 模板方法
        boilWater();
        brew();                // 直接调用虚函数
        pourInCup();
        addCondiments();
    }
};
```

### 2. NVI 模式（虚函数在private/protected）
```cpp
class CaffeineBeverage {
public:
    // 公有方法都是非虚的
    void prepareRecipe() {
        boilWater();
        brew();                // 调用私有虚函数
        pourInCup();
        addCondiments();
    }

private:
    // 虚函数放在私有区
    virtual void brew() = 0;
    virtual void addCondiments() = 0;
    
    void boilWater() { ... }
    void pourInCup() { ... }
};
```

---

## 🎯 关键区别
| 方面 | 普通模板模式 | NVI 模式 |
| --- | --- | --- |
| **虚函数位置** | 通常在 public | 强制在 private/protected |
| **接口设计** | 混合公有和虚函数 | 公有方法永远非虚 |
| **控制粒度** | 子类可直接重写 | 基类完全控制调用时机 |
| **前置/后置处理** | 不易添加 | 容易在基类添加（如日志、锁） |


---

## 💡 NVI 模式的核心优势
```cpp
class Base {
public:
    void process() {
        // 可以在调用前后添加通用逻辑
        cout << "前置处理" << endl;
        doProcess();  // 调用私有虚函数
        cout << "后置处理" << endl;
    }

private:
    virtual void doProcess() = 0;  // 子类只需关注核心逻辑
};
```

---

## 📌 结论
+ **模板模式**是一个更宽泛的概念
+ **NVI 模式**是模板模式的一种**特殊实现形式**，更严格、更安全
+ 可以认为：**NVI 是模板模式的最佳实践**

在 C++ 中，**推荐使用 NVI 模式**，因为它：

+ 更好地分离接口与实现
+ 更容易添加通用功能（日志、同步等）
+ 符合开闭原则
