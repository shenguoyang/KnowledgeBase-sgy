## 1. 业务场景：智能家居控制系统
想象你正在开发一个智能家居控制系统。这个系统需要控制多种设备：

+ **灯光系统**：开灯、关灯、调节亮度
+ **空调系统**：开关空调、设置温度、调整模式
+ **音响系统**：播放、暂停、切换歌曲
+ **窗帘系统**：打开、关闭、设置开合度

客户端需要能够：

+ 执行单个控制命令
+ 执行批量命令（场景模式）
+ 支持撤销/重做操作
+ 支持命令的延迟执行和排队

## 3. 代码问题分析
### 🔴 扩展性问题
**现象**：添加新设备需要修改多处代码

```cpp
// 添加音响设备时：
// 1. 需要在HomeController中添加新成员
// 2. 需要添加新的控制方法
// 3. 需要修改所有场景模式
class HomeController {
    // 需要添加
    // MusicPlayer* musicPlayer;
    // void controlMusicPlayer(...);
    
    // 需要修改所有场景方法
    void executeHomeScene() {
        // ... 现有代码
        // musicPlayer->play(); // 需要添加
    }
};
```

**实际后果**：随着产品迭代，控制器类会变得臃肿，每次新增设备都需要重新测试所有相关代码，开发周期延长。

### 🔴 耦合性问题
**现象**：控制器直接依赖具体设备类

```cpp
class HomeController {
    Light* light;          // 直接依赖具体类
    AirConditioner* ac;    // 直接依赖具体类
    // 添加新设备就需要添加新的依赖
};
```

**实际后果**：设备实现变更（如接口调整）会强制要求修改控制器代码，违背开闭原则，增加回归测试成本。

### 🔴 维护性问题
**现象**：需求变更（新增撤销功能，命令异步队列）导致大规模重构

```cpp

    // saveCurrentState();
```

**实际后果**：简单的功能增强需要修改大量已有代码，引入bug风险高。

### 🏷️ 各角色职责
1. **Command（命令接口）**
    - 抽象基类，声明`execute()`和`undo()`纯虚函数
    - 通常作为接口类（纯虚类）实现
2. **ConcreteCommand（具体命令）**
    - 继承Command，实现具体操作
    - 持有Receiver的引用（通常为weak_ptr避免循环引用）
    - C++中可使用模板实现通用命令
3. **Invoker（调用者）**
    - 持有命令对象（unique_ptr或shared_ptr）
    - 触发命令执行，可维护命令历史栈
    - 实现undo/redo功能
4. **Receiver（接收者）**
    - 实际执行业务逻辑的对象
    - 可以是任何已有的业务类
    - 不知道命令模式的存在
5. **Client（客户端）**
    - 创建具体命令对象
    - 设置命令的接收者
    - 将命令传递给调用者

## 5. 命令模式重构方案
```cpp
#include <iostream>
#include <memory>
#include <vector>
#include <stack>
#include <functional>

using namespace std;
class Light {
public:
    void turnOn()    
    void turnOff() 
    void setBrightness(int level)
};
class AirConditioner {
public:
    void turnOn()   
    void turnOff() 
    void setTemperature(int temp) 
};
// ==================== 命令接口 ====================
class Command {
public:
    virtual ~Command() = default;
    virtual void execute() = 0;
    virtual void undo() = 0;
    virtual string getName() const = 0;
};

// ==================== 具体命令类 ====================
// 灯光命令
class LightOnCommand : public Command {
public:
    explicit LightOnCommand(weak_ptr<Light> light) : light(light) {}
    
    void execute() override {
        if (auto l = light.lock()) {
            l->turnOn();
        }
    }
    
    void undo() override {
        if (auto l = light.lock()) {
            l->turnOff();
        }
    }
    
    string getName() const override { return "Light On"; }
    
private:
    weak_ptr<Light> light;
};

class LightOffCommand : public Command {
    // 类似实现，省略...
};

class SetBrightnessCommand : public Command {
public:
    SetBrightnessCommand(weak_ptr<Light> light, int level)
        : light(light), newLevel(level) {
            prevLevel = light->getCurrentBrightness();
        }
    
    void execute() override {
        if (auto l = light.lock()) {
            l->setBrightness(newLevel);
        }
    }
    
    void undo() override {
        if (auto l = light.lock()) {
            l->setBrightness(prevLevel);
        }
    }
    
    string getName() const override { 
        return "Set Brightness to " + to_string(newLevel);
    }
    
private:
    weak_ptr<Light> light;
    int newLevel;
    int prevLevel;
};

// 空调命令（类似实现）
class ACTurnOnCommand : public Command { /* ... */ };
class ACTurnOffCommand : public Command { /* ... */ };
class SetTemperatureCommand : public Command { /* ... */ };

// ==================== 批量命令 ====================
class MacroCommand : public Command {
public:
    MacroCommand(string name) : name(name) {}
    
    void addCommand(shared_ptr<Command> cmd) {
        commands.push_back(cmd);
    }
    
    void execute() override {
        cout << "执行宏命令: " << name << endl;
        for (auto& cmd : commands) {
            cmd->execute();
        }
    }
    
    void undo() override {
        cout << "撤销宏命令: " << name << endl;
        // 反向执行撤销
        for (auto it = commands.rbegin(); it != commands.rend(); ++it) {
            (*it)->undo();
        }
    }
    
    string getName() const override { return name; }
    
private:
    vector<shared_ptr<Command>> commands;
    string name;
};
// ==================== 命令控制器 ====================
class CommandController {
public:
    // 执行命令
    void executeCommand(shared_ptr<Command> cmd) {
        cmd->execute();
        history.push(cmd);
        cout << "执行命令: " << cmd->getName() << endl;
    }
    
    // 撤销
    void undo() {
        if (!history.empty()) {
            auto cmd = history.top();
            cmd->undo();
            history.pop();
            undoStack.push(cmd);
            cout << "撤销命令: " << cmd->getName() << endl;
        }
    }
    
    // 重做
    void redo() {
        if (!undoStack.empty()) {
            auto cmd = undoStack.top();
            cmd->execute();
            undoStack.pop();
            history.push(cmd);
            cout << "重做命令: " << cmd->getName() << endl;
        }
    }
    
    // 批量执行
    void executeCommands(const vector<shared_ptr<Command>>& cmds) {
        for (auto& cmd : cmds) {
            executeCommand(cmd);
        }
    }
    
    // 清空历史
    void clearHistory() {
        while (!history.empty()) history.pop();
        while (!undoStack.empty()) undoStack.pop();
    }
    
private:
    stack<shared_ptr<Command>> history;
    stack<shared_ptr<Command>> undoStack;
};

// ==================== 客户端调用 ====================
int main() {
    // 创建设备（使用智能指针管理）
    auto light = make_shared<Light>();
    auto ac = make_shared<AirConditioner>();
    
    // 创建命令控制器
    CommandController controller;
    
    // 创建具体命令
    auto lightOn = make_shared<LightOnCommand>(light);
    auto lightOff = make_shared<LightOffCommand>(light);
    auto setLight70 = make_shared<SetBrightnessCommand>(light, 70);
    
    auto acOn = make_shared<ACTurnOnCommand>(ac);
    auto acOff = make_shared<ACTurnOffCommand>(ac);
    auto setTemp23 = make_shared<SetTemperatureCommand>(ac, 23);
    
    // 创建宏命令（场景模式）
    auto homeScene = make_shared<MacroCommand>("家庭模式");
    homeScene->addCommand(lightOn);
    homeScene->addCommand(setLight70);
    homeScene->addCommand(acOn);
    homeScene->addCommand(setTemp23);
    
    // 执行命令
    cout << "=== 执行单个命令 ===" << endl;
    controller.executeCommand(lightOn);
    controller.executeCommand(setLight70);
    
    cout << "\n=== 执行宏命令 ===" << endl;
    controller.executeCommand(homeScene);
    
    cout << "\n=== 测试撤销功能 ===" << endl;
    controller.undo();  // 撤销家庭模式
    controller.undo();  // 撤销设置亮度
    controller.undo();  // 撤销开灯
    
    cout << "\n=== 测试重做功能 ===" << endl;
    controller.redo();  // 重做开灯
    controller.redo();  // 重做设置亮度
    controller.redo();  // 重做家庭模式
    
    // 命令队列示例
    cout << "\n=== 命令队列执行 ===" << endl;
    vector<shared_ptr<Command>> commandQueue = {lightOff, acOff};
    controller.executeCommands(commandQueue);
    
    return 0;
}
```

## 6. 解决方案优势分析
### 🔧 如何解耦对象关系
**改进前**：

```cpp
// 控制器直接操作设备
class HomeController {
    Light* light;  // 强耦合
    void controlLight() { light->turnOn(); }
};
```

**改进后**：

```cpp
// 通过命令接口解耦
class CommandController {
    stack<Command*> history;  // 只依赖抽象接口
    void executeCommand(Command* cmd) { cmd->execute(); }
};
```

**解耦原理**：控制器不再依赖具体设备，只与`Command`接口交互，符合**依赖倒置原则**。

### 🚀 如何提高扩展性 
**扩展新设备**：

```cpp
// 添加音响设备，无需修改控制器
class MusicPlayer {
    void play() { /* ... */ }
    void stop() { /* ... */ }
};

class MusicPlayCommand : public Command {
    void execute() override { player->play(); }
    void undo() override { player->stop(); }
    // 控制器可以直接使用此命令
};
```

**扩展原理**：通过**接口隔离变化点**，新增设备只需添加新的命令类，不修改已有代码。

### 📊 代码差异对比
```diff
// 重构前 - 硬编码场景

-     light->turnOn();
-     light->setBrightness(70);
-     ac->turnOn();
-     ac->setTemperature(23);
- }

// 重构后 - 动态组合命令

```

**改进核心**：将**过程式控制逻辑**转变为**声明式命令对象**，实现控制逻辑的数据化。

## 7. 命令模式核心总结
### 💎 核心思想
**一句话概括**：将请求封装为对象，使你可以参数化客户端使用不同的请求，队列化或记录请求，并支持可撤销的操作。

### 
### 🎯 典型应用场景
1. **需要撤销/重做功能**

参考第五章节

2. **需要命令队列或延迟执行**

```cpp
// 线程池任务调度
threadPool.addTask(make_shared<DatabaseCommand>(sql));
```

3. **事务处理和原子操作**

```cpp
// 数据库事务
auto transaction = make_shared<Transaction>();
transaction->addCommand(make_shared<InsertCommand>(data));
transaction->addCommand(make_shared<UpdateCommand>(data));
if (failed) transaction->undo();
```



### ⚠️ 反例场景（不适用的情况）
1. **简单的一次性操作**

```cpp
// 过度设计 - 不需要命令模式
auto cmd = make_shared<SimpleCommand>([](){
    cout << "Hello" << endl;
});
controller.executeCommand(cmd);

// 简单实现
cout << "Hello" << endl;
```

2. **性能敏感的实时系统**

```cpp
// 高频调用的游戏循环
void gameLoop() {
    // 每帧调用，命令对象创建开销大
    // auto cmd = make_shared<MoveCommand>(player); // 避免
    player.move();  // 直接调用
}
```

3. **命令间无状态依赖**

```cpp
// 如果不需要撤销、队列等功能
light.turnOn();  // 直接调用足够
```

4. **命令参数频繁变化**

```cpp
// 如果每次执行参数都不同
// 命令对象需要频繁创建，不如函数调用高效
for (int i = 0; i < 1000; ++i) {
    // 低效：每次创建新命令对象
    auto cmd = make_shared<CalculateCommand>(i);
}
```
