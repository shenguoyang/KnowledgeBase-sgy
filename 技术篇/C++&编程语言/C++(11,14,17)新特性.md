```cpp
[i]() -> int { return i * 2; };写法二

● [] 不捕获任何变量。
● [&] 捕获外部作用域中所有变量，并作为引用在函数体中使用（按引用捕获）。
● [=] 捕获外部作用域中所有变量，并作为副本copy在函数体中使用（按值捕获）。
● [bar]按值捕获bar变量，同时不捕获其他变量。
● [&bar]引用捕获bar变量，同时不捕获其他变量。
● [this] 捕获当前类中的 this 指针，让 lambda 表达式拥有和当前类成员函数同样的访问权限。如果已经使用了 & 或者 =，就默认添加此选项。捕获 this 的目的是可以在 lamda 中使用当前类的成员函数和成员变量
```

```cpp
#include <iostream>
#include <future>
#include <chrono>
#include <vector>

int task(int id) {
    std::cout << "Starting task " << id << std::endl;
    // 模拟任务需要一段时间才能完成
    std::this_thread::sleep_for(std::chrono::seconds(id));
    std::cout << "Task " << id << " completed" << std::endl;
    return id;
}

int main() {
    std::vector<std::future<int>> futures;
    // 启动四个线程，每个线程执行一个任务
    for (int i = 1; i <= 4; ++i) {
        futures.push_back(std::async(std::launch::async, task, i));
    }
    // 循环等待所有任务完成或者有一个任务完成
    while (true) 
    {
        for (auto& f : futures) 
        {
            if (f.valid() && f.wait_for(std::chrono::seconds(0)) == std::future_status::ready) 
            {
                // 有一个任务完成，输出相应的提示信息
                std::cout << "Task " << f.get() << " completed first" << std::endl;
                // 取消所有未完成的任务
                for (auto& f2 : futures) 
                {
                    if (f2.valid() && f2.wait_for(std::chrono::seconds(0)) == std::future_status::timeout) 
                    {
                        f2.wait();
                        f2.get();
                    }
                }
                return 0;
            }
        }
        // 等待一段时间后再次检查所有任务是否完成
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
    }
    return 0;
}
```

## 3.4 std::promise
**概念**：可以在线程间传递数据，可简单理解为线程运行完后发给其他线程的返回值。



```cpp
#include <iostream>
#include <future>
#include <thread>

void task(std::promise<int>& prom) {
    // 模拟一个耗时的任务
    std::this_thread::sleep_for(std::chrono::seconds(2));
    
    // 设置承诺对象的值
    prom.set_value(42);
    std::this_thread::sleep_for(std::chrono::seconds(2));
}

int main() {
    // 创建一个承诺对象和一个相关联的future对象
    std::promise<int> prom;
    std::future<int> fut = prom.get_future();
    
    // 在一个新线程中执行任务
    std::thread t(task, std::ref(prom));
    
    // 等待异步任务完成并获取结果
    int result = fut.get();
    
    // 输出结果
    std::cout << "The result is " << result << std::endl;
    
    // 等待线程结束
    t.join();
    
    return 0;
}

```


在这个例子中，**myPrint**函数接受任意数量和类型的参数，并且使用逗号分隔符输出它们。**Args**是一个参数包（parameter pack）类型，表示任意数量的参数。**args**也是一个参数包，用于接受传递给函数的参数。

在函数体内，我们使用了C++17中引入的折叠表达式（fold expression）来将所有参数连接起来。折叠表达式中的逗号表达式（comma expression）表示在顺序求值的情况下，先输出左边的表达式（**std::cout << args**），再输出右边的表达式，最后返回右边表达式的结果（即**std::cout**）。

### 类模板
// 递归调用展开参数包的形式

template<typename T, typename... Args>

class myTuple {

public:

    myTuple(T t, Args... args) : first(t), rest(args...) {}

    T first;

    myTuple<Args...> rest;

};



// 终止递归的形式

template<typename T>

class myTuple<T> {

public:

    myTuple(T t) : first(t) {}

    T first;

};



int main() {

    myTuple<int, std::string, double> t(1, "hello", 3.14);

    std::cout << t.first << " " << t.rest.first << " " << t.rest.rest.first << std::endl;

    return 0;

}



在这个例子中，**myTuple**类模板接受任意数量和类型的模板参数，表示一个元组（tuple）。类模板的定义分为两部分：一个是递归调用展开参数包的形式，另一个是终止递归的形式。

在递归调用的形式中，构造函数使用递归的方式，逐个接受传递给它的参数，然后调用一个具有更少参数的构造函数。**rest**成员变量是一个**myTuple**对象，表示元组中剩余的部分。这个过程一直递归到只有一个参数时，进入终止递归的形式。

在**main**函数中，我们创建了一个**myTuple<int, std::string, double>**类型的对象**t**，并输出其中的元素。这个过程通过递归调用**rest**成员变量来实现。

```cpp
	}
    //-------------------
    
    //原生数组
    int a[2] = {1, 2};
    auto [x, y] = a;
    auto& [xr, yr] = a;

    //std::tuple
	float aa = 0.0;
    std::tuple<float&, char&&, int> tpl(aa, std::move('a'), 0);
    auto&[aaa, bbb, ccc] = tpl;

    //c++17 Error
    //c++20 Ok
    for(const auto& [key, value]: map){
        [&key = key, &value = value]{
            std::cout << key << ": " << value << std::endl;
        }();
    }
    
    return 0;
}

```



**结论：**

**<font style="color:rgb(18, 18, 18);">结构化绑定是绑定到一个新的对象，由返回值进行初始化，而不是直接绑定到返回值本身。</font>**

****

**适用于：**

1、适用于所有只有 public 数据成员的结构体和类

2、C 风格数组

3、std::tuple、std::array、std::pair

****

**缺点： **

结构化绑定时声明的对象必须和绑定对象数量一致 。

所有的成员变量需要在子类或者父类中统一声明，不能出现父类和子类都有定义的情况。



![](https://cdn.nlark.com/yuque/0/2023/png/35088368/1675844662101-68ec87d9-5b84-4759-aeb1-33f99b5149ee.png)



**概念： **

主要针对lambda在struct/class中的应用，

c++17之前，labmbda当出现默认捕获时，都会隐式捕获当前对象，隐式捕获(不论是&/=)都为当前对象的引用捕获。(默认捕获&/=)

c++17，labmbda表达式可捕获*this，为当前对象的拷贝，以确保当前对象释放以后(如多线程情况下)，lambda表达式能够安全访问this中的变量和方法。



**示例1：**

```cpp
struct X
{
    int value = 100;

    auto getValueCaptureStarThis()
    {
        return [*this]{ return value; };
    }
    auto getValueCaptureThis()
    {
        return [&] { return value; };
    }
    auto getValueCaptureThis2()
    {
        return [=] { return value; };
    }
};

int main()
{
    X x;
    auto valueOne = x.getValueCaptureStarThis();
    auto valueTwo = x.getValueCaptureThis();
    auto valueThree = x.getValueCaptureThis2();
    x.value = 321;
    std::cout << valueOne() << " " << valueTwo() << " " << valueThree() << std::endl;
    return 0;
}
```

```cpp
class Work
```
