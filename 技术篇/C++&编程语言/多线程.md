# 流水线处理模式：场景、优势与实战应用
## 流水线模式的核心概念
流水线处理模式（Pipeline Processing）是一种将任务分解为多个**连续处理阶段**的设计模式，其中每个阶段专注于特定操作，数据像流水线一样依次通过各个处理单元。每个阶段由独立的处理器（线程/进程）执行，不同阶段可以**并行处理不同数据项**，从而最大化系统吞吐量。

```mermaid
graph LR
    A[原始数据] --> B[阶段1：数据清洗]
    B --> C[阶段2：特征提取]
    C --> D[阶段3：模型推理]
    D --> E[阶段4：结果输出]
```

## 适用场景与典型案例
### 1. 多媒体处理流水线（视频转码）
**场景描述**：

+ 需要将视频文件从一种格式转换为另一种格式
+ 包含多个处理阶段：解封装→解码→滤镜处理→编码→封装

**C++实现示例**：

```cpp
#include <iostream>
#include <vector>
#include <thread>
#include <queue>
#include <mutex>
#include <condition_variable>
#include <functional>

// 视频帧数据结构
struct VideoFrame {
    int frame_id;
    std::vector<uint8_t> data;
    // 其他元数据...
};

class VideoPipeline {
public:
    void start() {
        // 创建各阶段线程
        stages.emplace_back(&VideoPipeline::demux_stage, this);
        stages.emplace_back(&VideoPipeline::decode_stage, this);
        stages.emplace_back(&VideoPipeline::filter_stage, this);
        stages.emplace_back(&VideoPipeline::encode_stage, this);
        stages.emplace_back(&VideoPipeline::mux_stage, this);
    }

    void stop() {
        // 发送停止信号
        {
            std::unique_lock<std::mutex> lock(mtx);
            stop_flag = true;
        }
        cv.notify_all();
        
        // 等待所有线程结束
        for (auto& t : stages) {
            if (t.joinable()) t.join();
        }
    }

    void add_input(const std::string& file) {
        std::unique_lock<std::mutex> lock(mtx);
        input_queue.push(file);
        lock.unlock();
        cv.notify_one();
    }

private:
    std::vector<std::thread> stages;
    std::queue<std::string> input_queue;
    std::queue<VideoFrame> demux_queue;
    std::queue<VideoFrame> decode_queue;
    std::queue<VideoFrame> filter_queue;
    std::queue<VideoFrame> encode_queue;
    std::mutex mtx;
    std::condition_variable cv;
    bool stop_flag = false;

    // 解封装阶段：从文件提取视频流
    void demux_stage() {
        while (true) {
            std::string file;
            {
                std::unique_lock<std::mutex> lock(mtx);
                cv.wait(lock, [this] { 
                    return !input_queue.empty() || stop_flag; 
                });
                
                if (stop_flag && input_queue.empty()) return;
                if (input_queue.empty()) continue;
                
                file = input_queue.front();
                input_queue.pop();
            }
            
            // 模拟解封装处理
            std::cout << "解封装: " << file << "\n";
            std::this_thread::sleep_for(std::chrono::milliseconds(50));
            
            // 生成模拟帧数据
            for (int i = 0; i < 10; ++i) {
                VideoFrame frame{i, std::vector<uint8_t>(1024)};
                
                std::unique_lock<std::mutex> lock(mtx);
                demux_queue.push(frame);
            }
            cv.notify_one();
        }
    }

    // 解码阶段：将压缩数据解码为原始帧
    void decode_stage() {
        while (true) {
            VideoFrame frame;
            {
                std::unique_lock<std::mutex> lock(mtx);
                cv.wait(lock, [this] { 
                    return !demux_queue.empty() || stop_flag; 
                });
                
                if (stop_flag && demux_queue.empty()) return;
                if (demux_queue.empty()) continue;
                
                frame = demux_queue.front();
                demux_queue.pop();
            }
            
            std::cout << "解码帧: " << frame.frame_id << "\n";
            std::this_thread::sleep_for(std::chrono::milliseconds(30));
            
            {
                std::unique_lock<std::mutex> lock(mtx);
                decode_queue.push(frame);
            }
            cv.notify_one();
        }
    }

    // 滤镜阶段：应用视频效果
    void filter_stage() {
        while (true) {
            VideoFrame frame;
            {
                std::unique_lock<std::mutex> lock(mtx);
                cv.wait(lock, [this] { 
                    return !decode_queue.empty() || stop_flag; 
                });
                
                if (stop_flag && decode_queue.empty()) return;
                if (decode_queue.empty()) continue;
                
                frame = decode_queue.front();
                decode_queue.pop();
            }
            
            std::cout << "应用滤镜到帧: " << frame.frame_id << "\n";
            std::this_thread::sleep_for(std::chrono::milliseconds(40));
            
            {
                std::unique_lock<std::mutex> lock(mtx);
                filter_queue.push(frame);
            }
            cv.notify_one();
        }
    }

    // 编码阶段：将处理后的帧重新编码
    void encode_stage() {
        while (true) {
            VideoFrame frame;
            {
                std::unique_lock<std::mutex> lock(mtx);
                cv.wait(lock, [this] { 
                    return !filter_queue.empty() || stop_flag; 
                });
                
                if (stop_flag && filter_queue.empty()) return;
                if (filter_queue.empty()) continue;
                
                frame = filter_queue.front();
                filter_queue.pop();
            }
            
            std::cout << "编码帧: " << frame.frame_id << "\n";
            std::this_thread::sleep_for(std::chrono::milliseconds(35));
            
            {
                std::unique_lock<std::mutex> lock(mtx);
                encode_queue.push(frame);
            }
            cv.notify_one();
        }
    }

    // 封装阶段：将编码数据写入输出文件
    void mux_stage() {
        while (true) {
            VideoFrame frame;
            {
                std::unique_lock<std::mutex> lock(mtx);
                cv.wait(lock, [this] { 
                    return !encode_queue.empty() || stop_flag; 
                });
                
                if (stop_flag && encode_queue.empty()) return;
                if (encode_queue.empty()) continue;
                
                frame = encode_queue.front();
                encode_queue.pop();
            }
            
            std::cout << "写入帧到文件: " << frame.frame_id << "\n";
            std::this_thread::sleep_for(std::chrono::milliseconds(25));
        }
    }
};

int main() {
    VideoPipeline pipeline;
    pipeline.start();
    
    // 添加处理任务
    pipeline.add_input("video1.mp4");
    pipeline.add_input("video2.mov");
    pipeline.add_input("video3.avi");
    
    // 等待处理完成
    std::this_thread::sleep_for(std::chrono::seconds(3));
    pipeline.stop();
    
    return 0;
}
```

**好处**：

1. **高吞吐量**：当解封装处理第N+1个视频时，解码器可同时处理第N个视频的帧
2. **资源优化**：各阶段可使用不同硬件资源（如GPU加速解码/编码）
3. **弹性扩展**：可对瓶颈阶段（如滤镜处理）增加并行实例

## 何时避免使用流水线模式
1. **简单线性任务**：当任务没有明显阶段划分时
2. **超低延迟要求**：阶段间通信可能引入额外延迟
3. **强事务需求**：需要ACID保证的复杂事务处理
4. **小规模数据处理**：流水线开销可能超过收益

以下是四种异步任务协调机制在C++中的代码示例及其核心区别解析：

### 1. 屏障同步 (`std::barrier` - C++20)
```cpp
#include <iostream>
#include <thread>
#include <barrier>
#include <vector>

int main() {
    constexpr int N = 3;  // 3个工作线程
    std::barrier sync_point(N); // 创建屏障
    
    auto worker = [&](int id) {
        std::cout << "Worker " << id << " 阶段1完成\n";
        sync_point.arrive_and_wait();  // 等待所有线程到达
        
        std::cout << "Worker " << id << " 阶段2完成\n";
        sync_point.arrive_and_wait();  // 再次同步
    };

    std::vector<std::thread> threads;
    for (int i = 0; i < N; ++i) {
        threads.emplace_back(worker, i);
    }

    for (auto& t : threads) t.join();
}
/* 输出（每次运行顺序可能不同，但阶段会同步）：
Worker 0 阶段1完成
Worker 1 阶段1完成
Worker 2 阶段1完成
Worker 2 阶段2完成
Worker 0 阶段2完成
Worker 1 阶段2完成
*/
```

**核心区别**：所有线程必须到达同一个屏障点才能继续执行，**可重复使用**，线程角色平等。

---

### 2. 一次性事件 (`std::promise` + `std::future`)
```cpp
#include <iostream>
#include <thread>
#include <future>

int main() {
    std::promise<void> ready_promise;
    std::shared_future<void> ready_future = ready_promise.get_future().share();

    auto waiter = [&](int id) {
        ready_future.wait();  // 等待事件触发
        std::cout << "Worker " << id << " 开始工作\n";
    };

    std::thread t1(waiter, 1);
    std::thread t2(waiter, 2);

    std::cout << "初始化中..." << std::endl;
    std::this_thread::sleep_for(std::chrono::seconds(1));  // 模拟初始化
    ready_promise.set_value();  // 事件触发！

    t1.join(); t2.join();
}
/* 输出：
初始化中...
Worker 1 开始工作
Worker 2 开始工作
*/
```

**核心区别**：**单次触发不可重置**，一个设置者通知多个等待者，适用于初始化完成等一次性事件。

---

### 3. 条件变量 (`std::condition_variable`)
```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <condition_variable>
#include <queue>

std::mutex mtx;
std::condition_variable cv;
std::queue<int> data_queue;

void producer() {
    for (int i : {1, 2, 3}) {
        std::lock_guard<std::mutex> lk(mtx);
        data_queue.push(i);
        std::cout << "生产: " << i << std::endl;
        cv.notify_one();  // 通知一个等待者
    }
}

void consumer(int id) {
    while (true) {
        std::unique_lock<std::mutex> lk(mtx);
        cv.wait(lk, []{ return !data_queue.empty(); });  // 等待条件成立
        
        int data = data_queue.front();
        data_queue.pop();
        std::cout << "消费者" << id << " 消费: " << data << std::endl;
        lk.unlock();
        
        if (data == 3) break;  // 退出条件
    }
}

int main() {
    std::thread p(producer);
    std::thread c1(consumer, 1);
    std::thread c2(consumer, 2);
    
    p.join(); c1.join(); c2.join();
}
/* 输出示例：
生产: 1
消费者1 消费: 1
生产: 2
消费者2 消费: 2
生产: 3
消费者1 消费: 3
*/
```

**核心区别**：必须与锁配合使用，**等待自定义条件谓词**，可重复响应状态变化，适用于生产者-消费者模型。

---

### 4. 倒计时门闩 (`std::latch` - C++20)
```cpp
#include <iostream>
#include <thread>
#include <latch>
#include <vector>

int main() {
    constexpr int N = 5;  // 5个初始化任务
    std::latch start_latch(1);  // 主线程等待
    std::latch done_latch(N);   // 工作线程计数

    auto init_task = [&](int id) {
        std::cout << "任务" << id << " 初始化...\n";
        done_latch.count_down();  // 减少计数
        start_latch.wait();        // 等待主线程信号
        std::cout << "任务" << id << " 启动\n";
    };

    std::vector<std::thread> tasks;
    for (int i = 0; i < N; ++i) {
        tasks.emplace_back(init_task, i);
    }

    done_latch.wait();  // 等待所有初始化完成
    std::cout << "\n所有任务准备就绪！\n\n";
    start_latch.count_down();  // 释放所有任务

    for (auto& t : tasks) t.join();
}
/* 输出示例：
任务0 初始化...
任务1 初始化...
任务2 初始化...
任务3 初始化...
任务4 初始化...

所有任务准备就绪！

任务4 启动
任务0 启动
任务1 启动
任务2 启动
任务3 启动
*/
```

**核心区别**：**单次触发不可重置**，通过计数器机制等待指定数量的操作完成，工作线程与等待线程角色分离。

---

### 核心区别总结表
| 机制 | 触发条件 | 重用性 | 线程角色 | 典型场景 |
| --- | --- | --- | --- | --- |
| **屏障同步** | 所有线程到达屏障点 | 可重复使用 | 所有线程平等 | 多阶段并行计算 |
| **一次性事件** | 事件标志被设置 | 单次触发 | 设置者/等待者 | 全局初始化完成通知 |
| **条件变量** | 自定义条件谓词成立 | 可重复等待 | 通知者/等待者 | 生产者-消费者模型 |
| **倒计时门闩** | 计数器归零 | 单次触发 | 计数者/等待者 | 等待N个资源初始化完成 |


> 关键区别点：  
>
> 1. **屏障同步**强调所有线程在相同执行点汇合  
> 2. **一次性事件**关注事件发生的瞬时通知  
> 3. **条件变量**依赖共享状态的持续监测  
> 4. **倒计时门闩**通过计数机制协调任务完成状态
>

# 为什么需要 `std::promise`？与直接使用 `std::future` 的对比
`std::promise` 在 C++ 并发编程中扮演着关键角色，虽然 `std::future` 可以直接获取结果（如通过 `std::async`），但 `promise` 提供了更灵活、更底层的控制能力。以下是它们的主要区别和 `promise` 的必要性：

## 核心区别：角色定位
| 特性 | `std::future` | `std::promise` |
| --- | --- | --- |
| **角色** | 结果消费者（只读） | 结果生产者（只写） |
| **获取方式** | 从异步操作或 `promise` 获取 | 主动创建并设置值 |
| **控制权** | 被动等待结果 | 主动控制结果设置 |
| **灵活性** | 有限（依赖外部设置） | 高（可自定义设置点） |


## 为什么需要 `std::promise`？解决 `std::future` 的局限
### 1. 自定义结果设置点
**问题**：`std::async` 自动绑定任务结束点为结果设置点  
**解决**：`promise` 允许在代码任意位置设置结果

```cpp
// 使用 promise 在特定条件满足时设置结果
void complex_task(std::promise<int> prom) {
    if (initialization_failed()) {
        prom.set_exception(std::make_exception_ptr(std::runtime_error("Init failed")));
        return;
    }
    
    while (processing()) {
        if (found_result()) {
            prom.set_value(42); // 在中间点设置结果
            return;
        }
    }
    prom.set_value(0); // 最终设置结果
}

// 使用
std::promise<int> p;
std::future<int> f = p.get_future();
std::thread t(complex_task, std::move(p));
int result = f.get(); // 等待结果
```

### 2. 多结果源协调
**问题**：单个 `future` 只能关联一个结果源  
**解决**：多个线程可协作设置同一个 `promise`

```cpp
void partial_task(int id, std::promise<int>& prom, std::atomic<int>& counter) {
    int result = calculate_part(id);
    if (counter.fetch_sub(1) == 1) { // 最后一个完成的线程
        prom.set_value(result); // 设置最终结果
    }
}

// 使用
std::promise<int> final_result;
std::future<int> f = final_result.get_future();
std::atomic<int> remaining(3);

std::thread t1(partial_task, 1, std::ref(final_result), std::ref(remaining));
std::thread t2(partial_task, 2, std::ref(final_result), std::ref(remaining));
std::thread t3(partial_task, 3, std::ref(final_result), std::ref(remaining));

int total = f.get(); // 等待任意线程设置最终结果
```

### 3. 非返回值场景的结果传递
**问题**：无法通过返回值传递结果的情况  
**解决**：在回调函数、事件处理器中设置结果

```cpp
// 在回调函数中设置 promise
class Downloader {
    std::promise<std::string> download_promise;
public:
    void start_download(const std::string& url) {
        async_download(url, [this](const std::string& data) {
            download_promise.set_value(data); // 在回调中设置结果
        });
    }
    
    std::future<std::string> get_future() {
        return download_promise.get_future();
    }
};

// 使用
Downloader downloader;
auto fut = downloader.get_future();
downloader.start_download("https://example.com/data");
std::string data = fut.get(); // 等待下载完成
```

### 4. 高级同步模式
**问题**：`future` 只能表示简单的一次性事件  
**解决**：实现复杂同步原语

```cpp
// 使用 promise 实现一次性屏障
class SingleUseBarrier {
    std::promise<void> ready_promise;
    std::shared_future<void> ready_future;
public:
    SingleUseBarrier() : ready_future(ready_promise.get_future().share()) {}
    
    void signal_ready() {
        ready_promise.set_value(); // 解除所有等待
    }
    
    void wait() {
        ready_future.wait(); // 等待信号
    }
};

// 使用
SingleUseBarrier barrier;
std::thread worker([&] {
    do_work();
    barrier.signal_ready(); // 工作完成，通知所有等待者
});

// 多个等待线程
std::thread waiter1([&] { barrier.wait(); use_result(); });
std::thread waiter2([&] { barrier.wait(); use_result(); });
```

### 5. 异常处理的精确控制
**问题**：自动生成的 `future` 异常处理受限  
**解决**：精确控制异常设置点和类型

```cpp
void critical_section(std::promise<void> prom) {
    try {
        perform_unsafe_operation();
        prom.set_value();
    } catch (const std::exception& e) {
        // 创建包含丰富信息的异常
        prom.set_exception(std::make_exception_ptr(
            CriticalFailure("Operation failed at stage 3", e)
        ));
    } catch (...) {
        prom.set_exception(std::current_exception());
    }
}
```

## 关键场景总结：何时需要 `std::promise`
1. **自定义结果设置点** - 不在函数返回时设置结果
2. **多生产者协作** - 多个线程共同决定最终结果
3. **回调驱动架构** - 在异步回调中设置结果
4. **复杂同步原语** - 构建屏障、信号量等高级工具
5. **精确异常控制** - 定制异常类型和上下文信息
6. **非阻塞结果设置** - 使用 `set_value_at_thread_exit`
7. **与现有代码集成** - 包装传统回调接口为 future/promise

## 结论
虽然 `std::async` + `std::future` 提供了简单的异步编程模型，但 `std::promise` 才是异步结果传递机制的**底层基础**。它提供了：

+ **更精细的控制**：决定何时何地设置结果
+ **更大的灵活性**：适应各种异步模式和架构
+ **更强的表达能力**：实现复杂并发原语
+ **更精确的错误处理**：定制异常传播

当您需要超越简单任务返回值的异步控制时，`std::promise` 是必不可少的工具。它和 `std::future` 共同构成了 C++ 强大而灵活的异步编程基础。
