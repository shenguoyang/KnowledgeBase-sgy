# 线程安全：<font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">在多线程环境下，一个线程对共享资源的操作不会影响其他线程对该资源的操作</font>
线程同步：<font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">程同步是指在多线程环境下，通过一定机制使得多个线程对共享资源的操作按顺序执行，以保证线程安全</font>

## 1.1 互斥量
：只有拥有互斥对象的线程才有访问公共资源的权利

1.属于内核态

2.可不同进程的不同线程有效

3.需要内核切换，性能开销大

## 1.2 临界区（同步对象）
： 通过对多线程的串行化来访问公共资源，当有一个线程访问时，其他线程试图访问时会被挂起，属于进程维护对象

1.只针对同一个进程的不同线程有效

2.属于用户态，不属于内核态

3.linux没有临界区

4.不需要内核切换，所以性能开销小


## 1.3 信号量
：允许指定个数的线程访问公共资源

1.属于内核对象

示例1 两个线程任务交替运行

```cpp
#include <iostream>
#include <thread>
#include <semaphore.h>

std::semaphore sem1(1); // 初始值为1的信号量1
std::semaphore sem2(0); // 初始值为0的信号量2,表示一开始就阻塞线程，通过post可以增加1

void task1()
{
    for (int i = 0; i < 5; i++)
    {
        sem1.wait(); // 等待信号量1
        std::cout << "Task1 executing" << std::endl;
        sem2.post(); // 释放信号量2
    }
}

void task2()
{
    for (int i = 0; i < 5; i++)
    {
        sem2.wait(); // 等待信号量2
        std::cout << "Task2 executing" << std::endl;
        sem1.post(); // 释放信号量1
    }
}

int main()
{
    std::thread t1(task1);
    std::thread t2(task2);

    t1.join();
    t2.join();

    return 0;
}

```

## 1.5 条件变量：
```cpp
#include <iostream>
#include <queue>
#include <thread>
#include <mutex>
#include <condition_variable>

std::queue<int> q;
std::mutex mtx;
std::condition_variable cv;

void producer() {
    for (int i = 0; i < 10; i++) {
        std::unique_lock<std::mutex> lock(mtx);
        q.push(i);
        std::cout << "Produced: " << i << std::endl;
        lock.unlock();
        cv.notify_one();
    }
}

void consumer() {
    while (true) {
        std::unique_lock<std::mutex> lock(mtx);
        cv.wait(lock, [](){ return !q.empty(); });//当第二个参数位true时，wait会返回
        int val = q.front();
        q.pop();
        std::cout << "Consumed: " << val << std::endl;
    }
}

int main() {
    std::thread p(producer);
    std::thread c1(consumer);
    std::thread c2(consumer);

    p.join();
    c1.join();
    c2.join();

    return 0;
}

```

## 1.7 读写锁
```cpp
#include <iostream>
#include <thread>
#include <shared_mutex>

std::shared_timed_mutex mutex;
int counter = 0;

void increment() {
    std::unique_lock<std::shared_timed_mutex> lock(mutex);
    counter++;
    std::cout << "increment: " << counter << std::endl;
    lock.unlock();
}

void read() {
    std::shared_lock<std::shared_timed_mutex> lock(mutex);
    std::cout << "read: " << counter << std::endl;
    lock.unlock();
}

int main() {
    std::thread t1(increment);
    std::thread t2(increment);
    std::thread t3(read);

    t1.join();
    t2.join();
    t3.join();

    return 0;
}

```

# 2 线程同步选择策略


<font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">互斥量通常用于在单个共享资源上保护互斥访问。</font>

<font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">信号量通常用于在多个共享资源上保护互斥访问。</font>

<font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">条件变量通常用于在等待特定条件为真时同步线程。</font>

<font style="color:rgb(55, 65, 81);background-color:rgb(247, 247, 248);">读写锁通常用于保护共享资源的读写访问</font>
