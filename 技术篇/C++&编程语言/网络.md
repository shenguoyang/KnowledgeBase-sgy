以太网报头中前两个字段是以太网源地址和目的地址。目的地址为全1的特殊地址是广播地址。电缆上所有以太网接口都要接收广播数据帧。

协议类型字段表示要映射的协议地址类型，值为0x0800表示ip地址

硬件地址长度表示要mac地址长度，固定值为6 （字节），对应48bit

协议地址长度表示协议地址长度，当前值为4 （字节）， 对应ipv4的ip地址为32bit

操作（op）字段指出四种操作类型：

**<font style="color:rgb(51, 51, 51);">二、ARP缓存表详解</font>**

<font style="color:rgb(51, 51, 51);">上面我们讲解了交换机的工作原理，知道交换机是通过MAC地址通信的，但是我们是如何获得目标主机的MAC地址呢？这时我们就需要使用ARP协议了，在每台主机中都有一张ARP表，它记录着主机的IP地址和MAC地址的对应关系。</font>

<font style="color:rgb(51, 51, 51);">ARP协议：ARP协议是工作在网络层的协议，它负责将IP地址解析为MAC地址。</font>

<font style="color:rgb(51, 51, 51);">如下图：详细讲解ARP的工作原理。</font>

![](https://cdn.nlark.com/yuque/0/2022/jpeg/26460094/1658849765970-963a008d-10a7-4fcd-bf67-852ed30a4637.jpeg)

<font style="color:rgb(51, 51, 51);">1)如果主机A想发送数据给主机B，主机A首先会检查自己的ARP缓存表，查看是否有主机B的IP地址和MAC地址的对应关系，如果有，则会将主机B的MAC地址作为源MAC地址封装到数据帧中。如果没有，主机A则会发送一个ARP请求信息，请求的目标IP地址是主机B的IP地址，目标MAC地址是MAC地址的广播帧(即FF-FF-FF-FF-FF-FF)，源IP地址和MAC地址是主机A的IP地址和MAC地址。</font>

<font style="color:rgb(51, 51, 51);">2)当交换机接受到此数据帧之后，发现此数据帧是广播帧，因此，会将此数据帧从非接收的所有接口发送出去。</font>

<font style="color:rgb(51, 51, 51);">3）当主机B接受到此数据帧后，会校对IP地址是否是自己的，并将主机A的IP地址和MAC地址的对应关系记录到自己的ARP缓存表中，同时会发送一个ARP应答，其中包括自己的MAC地址。</font>

<font style="color:rgb(51, 51, 51);">4)主机A在收到这个回应的数据帧之后，在自己的ARP缓存表中记录主机B的IP地址和MAC地址的对应关系。而此时交换机已经学习到了主机A和主机B的MAC地址了。</font>

DHCP（Dynamic Host Configuration Protocol，动态主机配置协议）是一个网络协议，用于自动分配IP地址和其他网络参数给网络中的设备。这样，设备无需手动配置IP地址和其他相关设置，就可以加入网络并与其他设备进行通信。DHCP协议定义了一种客户端-服务器模型，其中DHCP客户端请求网络配置信息，而DHCP服务器负责提供这些信息。



DHCP的工作原理可以简化为以下四个步骤，这四个步骤通常被称为DORA过程（Discover, Offer, Request, Acknowledge）：



1.  Discover（发现）：当一个设备（如计算机、手机等）加入网络时，它会作为DHCP客户端发送一个DHCP Discover广播包。这个广播包的目的是寻找网络中的DHCP服务器，并请求分配一个IP地址。 
2.  Offer（提供）：收到DHCP Discover广播包的DHCP服务器会为客户端分配一个未被使用的IP地址，并将其与其他网络参数（如子网掩码、网关、DNS服务器等）一起封装在一个DHCP Offer包中，然后发送给DHCP客户端。如果网络中有多个DHCP服务器，客户端可能会收到多个DHCP Offer包。 
3.  Request（请求）：DHCP客户端收到一个或多个DHCP Offer包后，会选择其中一个（通常是第一个收到的Offer），并发送一个DHCP Request包给相应的DHCP服务器，以请求使用该服务器提供的IP地址和网络参数。 
4.  Acknowledge（确认）：DHCP服务器收到DHCP Request包后，会发送一个DHCP Ack包给客户端，确认分配的IP地址和网络参数。此时，DHCP客户端会根据这些信息配置自己的网络接口，并加入网络。如果分配的IP地址有问题，DHCP服务器可能会发送一个DHCP Nak包给客户端，表示拒绝请求。 



需要注意的是，DHCP分配的IP地址通常具有一个租约期（Lease Time），在租约期到期前，客户端需要重新请求延长租约。如果设备在租约期内离开网络，它应该向DHCP服务器发送一个DHCP Release包，以释放占用的IP地址，从而让其他设备可以使用该IP地址。



通过使用DHCP协议，网络管理员可以轻松地管理和分配IP地址，无需手动为每个设备配置网络参数。这大大简化了网络管理和设备接入的过程，提高了网络的可扩展性和可维护性。











路由器两个重要的模式介绍

1. 上网模式（WAN或Internet设置）

上网模式是指路由器如何连接到互联网或上游网络。这涉及到路由器的WAN口（或Internet口）的设置，以及与Internet服务提供商（ISP）的通信。常见的上网模式有动态IP（DHCP）、静态IP、PPPoE等。选择适当的上网模式取决于您的网络环境和ISP的要求。

+ 动态IP（DHCP）：路由器自动从上游设备（如调制解调器或光猫）获取IP地址、子网掩码、默认网关和DNS服务器信息。
+ 静态IP：需要手动输入IP地址、子网掩码、默认网关和DNS服务器信息，这些信息通常由ISP提供。
+ PPPoE：适用于需要用户名和密码进行身份验证的宽带连接。需要输入从ISP获取的用户名和密码来建立连接。
2. 连接模式（路由器的工作模式）

连接模式是指路由器在网络中的工作模式。根据不同的应用场景，路由器可以充当不同角色，如路由器模式、无线中继（桥接）模式、接入点模式、客户端模式等。



+ 路由器模式：这是路由器的默认工作模式，它提供路由、NAT、DHCP服务器等功能，将内部网络（LAN）与外部网络（WAN）连接起来。
+ 无线中继（桥接）模式：路由器通过无线连接将两个或多个网络桥接在一起，扩展无线覆盖范围。
+ 接入点模式：路由器仅作为一个无线接入点，提供无线连接。在此模式下，路由器的路由功能和DHCP服务器功能被禁用，设备通过路由器的无线连接加入现有的有线网络。
+ 客户端模式：路由器充当无线客户端，通过无线连接加入另一个无线网络。在此模式下，路由器的路由功能和DHCP服务器功能被禁用。



总之，上网模式主要涉及路由器如何连接到互联网或上游网络，而连接模式则是指路由器在网络中的工作模式。配置适当的上网模式和连接模式可以确保路由器根据您的需求和网络环境正确工作。
