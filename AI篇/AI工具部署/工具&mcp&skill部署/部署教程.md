## 2.2 cc-visualizer 部署流程与环境依赖

cc-visualizer 的设计理念是 **"零配置"**——绝大多数情况下只需一条命令即可运行，无需手动安装复杂依赖。


### 🚀 核心部署流程（推荐方式）

**一条命令启动：**
```bash
npx cc-visualizer
```

这条命令会自动完成以下三个操作：
1. **安装 Claude Code hooks**：自动写入 `~/.claude/settings.json`（PreToolUse、PostToolUse、Notification、Stop、SubagentStop 事件）
2. **启动可视化服务器**：默认监听端口 `7331`
3. **打开浏览器**：自动访问 `http://localhost:7331`

之后正常运行 `claude` 命令，所有事件会自动实时流入可视化界面。


### 📋 环境依赖清单

#### 运行时依赖（必需）

| 依赖项 | 版本要求 | 说明 |
|:---|:---|:---|
| **Node.js** | ≥18.0.0（建议最新 LTS） | 运行 `npx` 和 cc-visualizer 的核心环境 |
| **npm** | 随 Node.js 一起安装 | 用于下载和执行 npx 包 |
| **Claude Code** | 已安装并配置 | 可视化工具需要拦截 Claude Code 的事件 |

#### 可选依赖（按需）

| 依赖项 | 使用场景 |
|:---|:---|
| **现代浏览器**（Chrome/Edge/Firefox/Safari 最新版） | 打开 `http://localhost:7331` 查看可视化界面，需要 WebGL 支持 3D 渲染 |
| **Git**（仅源码部署） | 需要克隆源码仓库时使用 |

#### 硬件/网络要求

| 项目 | 最低要求 | 说明 |
|:---|:---|:---|
| **端口** | 7331（可自定义） | 确保该端口未被占用 |
| **网络** | 本地回环（localhost） | 无需互联网连接，所有数据本地运行 |
| **内存** | ~500MB | 主要用于 Node.js 服务和浏览器页面 |


### 🔧 启动选项说明

```bash
npx cc-visualizer --port=4000   # 自定义端口
npx cc-visualizer --no-open     # 不自动打开浏览器
npx cc-visualizer --no-hooks    # 仅启动服务器，不安装 hooks
npx cc-visualizer --uninstall   # 卸载 cc-visualizer 的 hooks 配置并退出
```


### 📦 源码部署流程（可选）

如需开发或调试源码，可按以下步骤部署：

```bash
# 1. 克隆仓库（将 your-username 替换为实际仓库地址）
git clone https://github.com/your-username/cc-visualizer
cd cc-visualizer

# 2. 安装依赖
npm install

# 3. 启动开发服务器
npm run dev
```

开发服务器同样启动在 `http://localhost:7331`。如需发送测试事件而不运行 Claude Code，可以向 `/api/hooks` 发送 POST 请求，JSON 格式需匹配 `lib/types.ts` 中定义的 `ClaudeEvent` 结构。


### ⚙️ Hook 工作原理

cc-visualizer 通过在 Claude Code 的 `settings.json` 中为每个 hook 事件追加一条 `curl` 命令来工作：

```bash
curl -sf -X POST http://localhost:7331/api/hooks \
  -H 'Content-Type: application/json' \
  --data-binary @- --max-time 2 2>/dev/null || true
```

**关键特性：**
- **非破坏性追加**：在现有 hooks 基础上追加，不覆盖已有配置
- **超时保护**：`--max-time 2` 确保 curl 最多等待 2 秒
- **不阻塞 Claude**：`|| true` 保证即使可视化工具未运行，Claude Code 也能正常工作


### ✅ 环境检查清单

部署前请确认：

- [ ] 已安装 Node.js 18+（`node -v` 检查）
- [ ] 已安装 Claude Code（`claude --version` 检查）
- [ ] 端口 7331 未被占用（`lsof -i :7331` 检查，或自定义端口）
- [ ] 终端有权限写入 `~/.claude/settings.json`

如果上述条件都满足，直接运行 `npx cc-visualizer` 即可看到可视化界面。
